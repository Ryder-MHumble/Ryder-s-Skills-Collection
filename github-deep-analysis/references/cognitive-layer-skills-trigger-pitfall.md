# 技能触发认知层失效陷阱

## 现象描述

当技能（skill）触发时，AI助手容易进入"认知已闭环"状态，注意力从"这是不是正确的问题"转移到"如何执行skill预设流程"。这种机制性偏差会导致在错误方向上高效执行。

## 典型案例

### 案例：2026-05-14 CloakBrowser分析

**用户请求**：
> "帮我深度解析一下：https://github.com/CloakHQ/CloakBrowser这个项目，当前市面上这么多的浏览器自动化工具，这个有啥不一样"

**问题分析**：
1. 用户话语包含明显对比信号："市面上这么多的浏览器自动化工具，这个有啥不一样"
2. github-deep-analysis技能被触发
3. 注意力转移到技能预设流程：获取项目元数据、浏览主页、准备全维度战略分析
4. **认知层失效**：未执行升维优先检查"这是不是正确的问题"，未确认用户真实意图是比较不同工具的核心差异（对比分析）还是全面战略分析
5. 技能预设路径覆盖了用户真实意图识别

## 认知层保护机制

### 1. 技能触发警报系统

当检测到以下信号时，必须暂停技能执行流程，先执行认知层复查：

| 高风险信号 | 含义 | 应对动作 |
|-----------|------|----------|
| "市面上有.../有什么不同" | 对比意图明确 | 立即确认：用户需要对比分析还是深度分析 |
| "和...相比" | 比较意图 | 澄清比较对象和维度 |
| "哪个更好/更合适" | 选择决策 | 转为决策支持框架 |
| 模糊主请求+对比信号 | 意图有歧义 | 必须确认主导意图 |

### 2. 复查决策树

```
用户请求 → 技能匹配 → 认知层复查节点：
    ↓
是否包含对比/比较/选择信号？
    ├─ 是 → "对比信号检测到，用户需要对比分析还是完整深入分析？"
    └─ 否 → 按技能预设路径执行
```

### 3. 升维检查清单（技能触发后必须执行）

- [ ] 技能预设路径与用户话语中的显性信号是否一致？
- [ ] 是否存在被技能关键词掩盖的隐式意图？（如"深度解析"掩盖"对比"）
- [ ] 用户真实意图是探索、决策、评估还是比较？
- [ ] 如果用户意图有歧义，是否应该先澄清再执行？

## 预防措施

### 1. 技能设计层面

- 在技能触发条件中明确对比分析路径
- 在技能开头添加"认知层复查提醒"
- 建立对比分析框架与战略分析框架的明确切换点

### 2. 执行层面

- 技能触发≠认知闭环，必须在执行预设流程前复查意图一致性
- 对比信号权重高于模糊分析信号（如"深度解析" vs "有什么不一样"）
- 当有疑问时，宁可多问一个确认问题，不要默认路径

### 3. 用户偏好识别

用户对技能触发过度执行反应敏感：
- 反感"看到关键词就自动匹配技能预设路径"
- 期望AI能识别话语中的多重信号并优先响应最强信号
- 对"在错误方向上高效执行"零容忍

## 集成到分析工作流

在github-deep-analysis技能中，对比分析框架已建立（见contrast-analysis-framework.md），但需要确保：

1. **触发优先级**：对比信号 > 深度分析信号
2. **确认机制**：当对比信号与模糊分析信号共存时，主动确认
3. **双框架处理**：对比分析可作为深度分析的补充章节，或作为独立报告

## 关键规则

1. **技能≠自动导航**：技能是工具箱，不是自动驾驶，用户意图是方向盘
2. **信号权重排序**：显性对比信号 > 模糊分析信号
3. **宁可询问不沉默**：当意图不明时，宁可多问一句，不要默认选择技能预设路径
4. **历史教训固化**：2026-05-14案例必须作为技能触发认知层失效的典型教训嵌入所有分析类技能

## 应用扩展

这一陷阱不仅存在于github-deep-analysis技能，任何触发式技能都存在类似风险：
- 文档处理技能可能忽略用户的实际使用场景
- 代码分析技能可能过度关注技术细节而忽略商业价值
- 评估类技能可能套用固定框架而忽略特定上下文

**解决原则**：技能触发后必须保持认知层第1条约束（升维优先）的优先级，在走预设流程前复查"这个预设路径是否匹配用户真实意图"。

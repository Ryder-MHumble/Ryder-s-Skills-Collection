# KOL 访谈情报引擎

> YouTube KOL 访谈 → 结构化深度分析 → 飞书 Wiki 文档，全自动一条龙。

## 解决什么问题

AI/科技领域每天产出大量高质量视频访谈，但看完一个 60 分钟的视频需要 60 分钟，看完之后能记住多少？一周后还剩多少？

这个 Skill 把「看视频 → 提炼洞察 → 沉淀文档」这件事自动化了。不是摘要，是**带原文佐证的深度分析**——每个核心判断都附有时间戳定位的逐字稿原文，可追溯、可验证。

## 核心能力

| 能力 | 说明 |
|------|------|
| **智能扫描** | 每日自动扫描信源频道，三层漏斗过滤（大佬命中→关键词命中→跳过），只处理有价值的视频 |
| **深度分析** | 不是观点清单流水账，按内容流分章节，章节标题从矛盾中产生，叙事有弧线 |
| **原文佐证** | 每个核心判断附带带时间戳的逐字稿原文（不翻译），可回溯定位到视频具体位置 |
| **自动入库** | 分析结果自动写入飞书 Wiki，结构化排版（标题/引用/代码块/分隔线），开箱即读 |
| **手动+定时双模式** | 给 URL 即刻分析；或设定时任务每天自动扫描信源频道，无人工干预 |

## 使用场景

**场景 1：看到好视频，即刻产出**
> 发一条 YouTube 链接，5 分钟后拿到一份带原文佐证的深度分析文档，自动存入飞书 Wiki。

**场景 2：每日情报巡检，零人工**
> 设定定时任务，每天自动扫描 13+ 信源频道，发现大佬访谈自动跑完全流程（扫描→获取逐字稿→深度分析→写 Wiki），早上打开飞书就有新情报。

**场景 3：团队情报共享**
> 分析文档自动写入飞书 Wiki 指定节点，团队成员无需看视频即可获取核心洞察，原文佐证支持深度追溯。

## 工作流程

```
信源频道（YouTube）                    飞书 Wiki
  │                                      │
  ├─ 自动扫描 ──→ 三层漏斗过滤 ──→ 深度分析 ──→ 自动写入
  │               (Tier 1/2/3)       (章节+佐证)    (结构化文档)
  │
  └─ 手动给 URL ──────────────────→ 深度分析 ──→ 自动写入
```

### 三层漏斗

| 层级 | 条件 | 动作 |
|------|------|------|
| 🔴 Tier 1 | 大佬名字命中（≥5min，非新闻评论） | 自动跑全流程，优先级最高 |
| 🟡 Tier 2 | AI 关键词命中 OR 高信噪比频道+长视频(≥15min) | 自动跑全流程，优先级次之 |
| ⚪ Tier 3 | <5min / 无命中 | 跳过 |

每日全流程上限 3 个，超出部分自动进入待处理队列，次日优先处理。

## 产出样例

每份文档包含：

- **核心判断**（引用块）：一句话概括整场访谈的赌注
- **4-6 个分析章节**：标题从内容矛盾中产生，叙事有弧线
- **逐字稿原文佐证**（代码块）：带时间戳 `[MM:SS]`，不翻译原文
- **元信息**：标题、频道、发布时间、时长、链接

```
┌─────────────────────────────────────────┐
│ 💬 核心判断一句话（含赌注描述）            │
│    —— KOL名, 来源+年份                    │
├─────────────────────────────────────────┤
│ 一、章节标题（从矛盾中产生）                │
│   1.1 子标题                             │
│   分析段落（中文）                        │
│   [MM:SS] 逐字稿原文（不翻译）             │
│   ...                                   │
├─────────────────────────────────────────┤
│ 元信息 / 标题 / 频道 / 时长 / 链接         │
└─────────────────────────────────────────┘
```

## 快速开始

### 手动模式

向你的 AI Agent 发送 YouTube 视频链接，要求分析即可：

```
帮我分析这个视频：https://www.youtube.com/watch?v=XXXXX
```

### 定时模式

配置 cron job，每天自动扫描信源频道：

```
0 10 * * *  →  加载 skill → 执行模式 B 全流程
```

扫描信源频道列表、关键词、大佬名单均可自定义（修改 `scripts/scan_youtube.py`）。

## 关键设计决策

| 决策 | 理由 |
|------|------|
| 逐字稿原文不翻译 | 英文原文可追溯、可验证，翻译会丢失语境和准确性 |
| 章节标题从矛盾中产生 | 观点清单流水账是信息降维，矛盾驱动的标题才能还原思维张力 |
| 每日上限 3 个 | 防止 cron 超时和 API 限流，超出部分自动排队次日处理 |
| Code block 强制 `wrap: true` | 逐字稿原文较长，不换行则单行溢出不可读 |

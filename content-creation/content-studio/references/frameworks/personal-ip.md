# 个人 IP 打造框架

> **核心**：IP 不是"内容多"，是"人格稳"。
> 一次成功 = 偶然。持续稳定 = IP。

## 1. IP 是什么 vs 不是什么

| IP 是 | IP 不是 |
|-------|--------|
| 一个人格的持续呈现 | 一个产品的功能介绍 |
| 一组价值观的反复表达 | 一次爆款事件的流量收割 |
| 一种生活方式的代言 | 单纯的"日更" |
| 用户的"身份认同"投射 | 用户的"工具使用"关系 |

**判断标准**：用户关注你是因为"想成为你"还是"想用你的工具"？
- 想成为 → IP
- 想用 → 产品

## 2. IP 三要素（缺一不可）

### 2.1 人设（Persona）

**人设不是假装的性格，是放大的真实**。

| 维度 | 问题 |
|------|------|
| 我是谁？ | 身份 / 职业 / 经历 |
| 我信什么？ | 价值观 / 立场 / 反共识 |
| 我用什么语言？ | 表达风格 / 口头禅 / 词汇偏好 |
| 我反对什么？ | 反对者画像（让自己鲜明） |
| 我的故事？ | 个人经历 / 转折点 / 失败 / 胜利 |

**Ryder 默认人设**（来自 user profile）：
- 身份：AI 产品经理
- 风格：直白、克制、专业、有温度
- 反对：油腻话术、空洞 AI 概念、过度营销
- 信条：「代码里读生意」（隐性立场）
- 受众：对 AI 感兴趣，不限技术深度

### 2.2 定位（Niche）

**不要做大类，要做交叉点**。

| 错误 | 正确 |
|------|------|
| "我是 AI 博主" | "我是 AI 产品经理的'反 AI 焦虑'专家" |
| "我做内容运营" | "我帮独立开发者做'零成本冷启动'" |
| "我是设计师" | "我是 B2B SaaS 的'工程师友好设计'布道者" |

**定位公式**：
```
定位 = [身份] + [垂直场景] + [独特视角/反共识]
```

### 2.3 视觉识别（Visual Identity）

**视觉 = 用户认识你的最快路径**。

| 元素 | 你的选择 |
|------|----------|
| 头像 | 与定位一致（不要换） |
| 主色 | 1-2 种（贯穿所有平台） |
| 字体 | 1-2 种（贯穿所有平台） |
| 卡片风格 | 选 1 种（详见 `references/styles/`） |
| 标志性元素 | 1 个（emoji / 符号 / 数字） |

**Ryder 默认视觉**（推荐）：
- 主色：深蓝 #1F4E79（专业感）
- 字体：Inter / 思源黑体
- 卡片风格：**McKinsey Business**（克制专业）
- 标志性元素：🟠（来自 WorkSelfie case study 的延续）

## 3. IP 内容矩阵（4 类内容长期组合）

**不要只做一类内容**，否则会疲劳或同质化。

| 类别 | 占比 | 例子 |
|------|------|------|
| **价值观输出** | 30% | "为什么我不做 AI 日报" / "AI 焦虑是没必要的" |
| **专业洞察** | 30% | 行业分析 / 工具评测 / 趋势预判 |
| **个人故事** | 20% | 项目复盘 / 失败教训 / 转折点 |
| **互动 / 福利** | 20% | Q&A / 资源包 / 工具分享 |

**为什么这个比例**：
- 价值观 → 让用户"想成为你"
- 洞察 → 让用户"想听你讲"
- 故事 → 让用户"信任你"
- 互动 → 让用户"记得你"

## 4. 内容生产节奏

### 4.1 单平台（小红书）

| 频次 | 状态 |
|------|------|
| 日更 | **不推荐**（消耗大，易疲） |
| 周更 2-3 | **推荐**（质量优先） |
| 周更 1 | **最少**（保持存在感） |

### 4.2 跨平台（X + 小红书 + 抖音 + B 站）

| 频次 | 建议 |
|------|------|
| 1 主平台深度（小红书 / X） | 2-3 篇/周 |
| 1 视频平台（抖音 / B 站） | 1 条/周 |
| 其他平台 | 同内容改写（不原创） |

**注意**：跨平台**不要 1 份内容复制**，要**1 份内容改写**（见 `references/platforms/*.md`）

## 5. IP 增长飞轮

```
高质量内容（人格 + 洞察）
    ↓
精准受众（关注）
    ↓
互动 / 转发（信任）
    ↓
回头客 / 社群（粘性）
    ↓
付费 / 影响力变现
    ↓
更高质量内容（更精准受众）
```

**飞轮关键**：从"内容→关注"到"关注→社群"是关键跳跃。

## 6. IP 失败模式（不要做）

| 失败 | 原因 |
|------|------|
| 1 周爆款后没下文 | 飞轮断 |
| 日更但都是"水文" | 信任崩 |
| 风格多变 | 没人记得你 |
| 强推产品 | 变成"营销号"，不是 IP |
| 拒绝商业化 | 飞轮转不动 |
| 商业化太狠 | 信任崩 |
| 没人设 | 像机器人发的 |
| 没有反共识 | 像所有 AI 博主 |

## 7. IP 自检清单（每月问自己）

- [ ] 我过去 30 天发了什么类型内容？（4 类齐了吗？）
- [ ] 用户的评论里有没有"想成为你"的话？
- [ ] 我的头像 / 主色 / 风格是不是 30 天没变过？
- [ ] 你的反对者是谁？清楚吗？
- [ ] 你的标志性梗 / 元素是什么？能立刻说出来吗？
- [ ] 你的 3 个核心价值观是什么？每条内容都体现吗？

## 8. 与其他 skill 的关系

| skill | 关系 |
|-------|------|
| `multi-platform-content` | **前身**，本 skill 的内容方法论从这里继承 |
| `brandkit` | 视觉资产（logo / 配色 / 规范）→ 长期用 |
| `self-distill` | 自我画像提炼 → 帮你找到人设 |
| `humanizer` | 文字去 AI 味 → 写完后用 |

## 9. IP 启动 SOP（从 0 到 1）

```
Step 1：定义（人设 + 定位 + 视觉）
   ↓
Step 2：选 1 主平台（推荐：小红书 / X）
   ↓
Step 3：发 10 条测试内容（1 个月内）
   ↓
Step 4：看反馈（哪种内容互动最好）
   ↓
Step 5：放大优势内容（增加频次）
   ↓
Step 6：扩张到 2 平台（+ 1 视频平台）
   ↓
Step 7：建立内容矩阵（4 类内容）
   ↓
Step 8：跨平台扩张（4 平台齐发）
   ↓
Step 9：商业化 / 社群 / 影响力变现
```

**关键**：Step 1-2 必须做扎实，不要跳到 Step 4。

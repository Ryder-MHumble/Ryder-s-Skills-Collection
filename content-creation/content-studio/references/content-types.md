# 4 种内容类型 × 4 平台适配方法论

> Ryder 的 4 种核心内容：AI产品推荐 / GitHub深度分析 / 热点事件讲解 / 日常感悟。
> 每种内容类型有**不同结构、不同钩子、不同平台适配**。
> 用法：先选内容类型 → 再选平台 → 查对应结构模板。

## 速查矩阵

| 内容类型 | 主钩子 | 小红书 | X | 抖音 | B站 |
|---------|--------|--------|---|------|-----|
| AI产品推荐 | 对比冲击 | 痛点→产品→体验 | Indie Hacker / OS Announcement | 录屏口播30s | 深度评测5min |
| GitHub深度分析 | 圈层暗号 | 图文卡片5张 | Thread 4条 | 3要点速览60s | 长视频10min |
| 热点事件讲解 | 反认知 | 快速图文 | 独狼帖 | 口播60s | 深度分析8min |
| 日常感悟 | 生活方式 | 图文1张 | One-liner | 口播30s | — |

## 1. AI产品推荐

> **本质**：不是"介绍产品"，是"替用户做选择"。用户不关心产品多好，关心**我用了能不能变强**。

### 结构模板

```
[痛点场景 1-2 句] → 这个产品怎么解决的 → 核心亮点 3 个 → 我的真实体验 → CTA
```

### 平台适配

| 平台 | 形式 | 结构 | 钩子 | 字数/时长 |
|------|------|------|------|----------|
| 小红书 | 图文 3-5 张 | 封面：产品名+一句话价值 / 正文：痛点→3亮点→体验 | 对比冲击 | ≤300字 |
| X | 独狼帖 | `Built [产品] that [能力]. No [痛点]. Just [价值]. link` | 对比冲击 | ≤280字符 |
| 抖音 | 录屏+口播 | 黄金3秒（"用了这个我回不去了"）→ 录屏3个操作 → CTA | 痛点共鸣 | 30-45s |
| B站 | 深度评测 | 开场（"今天评测XX"）→ 功能演示 → 对比竞品 → 适合谁 → CTA | 反认知 | 5-8min |

### 关键原则

- **推别人的产品**：先说"为什么不是我自己做的"（建立客观性），再说产品亮点
- **推自己的产品**：先说痛点（"我自己遇到了这个问题"），再说解决方案
- **永远不说**"怎么用"（产品文档不是社媒内容）
- **永远带**1 个具体数字（"省了2小时"比"提高效率"强100倍）

### 小红书标题公式

- `[数字]个让我后悔没早用的AI工具`
- `用了[数字]个AI工具后，我把[传统方式]戒了`
- `[产品名] vs [竞品名]：[数字]天实测对比`
- `打工人必备！这个AI工具让我[具体收益]`

## 2. GitHub深度分析

> **本质**：不是"翻译 README"，是"借项目作切片，看技术趋势和产品机会"。详见 `github-deep-analysis` skill。

### 结构模板

```
[一句话定位项目] → 技术亮点 3 个 → 产品视角洞察 → 适用场景 → GitHub 链接
```

### 平台适配

| 平台 | 形式 | 结构 | 钩子 | 字数/时长 |
|------|------|------|------|----------|
| 小红书 | 图文 5-9 张 | 封面：项目名+一句话 / 内页：架构图+亮点+洞察 | 圈层暗号 | ≤300字 |
| X | Thread 4条 | 1/n 一句话定位 → 2/n 技术亮点 → 3/n 产品洞察 → 4/n 链接 | 圈层暗号 | 4×280字符 |
| 抖音 | 口播+截图 | 3秒（"这个GitHub项目炸了"）→ 3个亮点截图 → 链接在评论区 | 反认知 | 45-60s |
| B站 | 长视频 | 开场 → 项目背景 → 技术拆解 → 产品视角 → 适用人群 → CTA | 圈层暗号 | 8-12min |

### 关键原则

- **不说**"这个项目很厉害"（空话）
- **说**"这个项目解决了[具体问题]，用[具体技术]，比[竞品]快[数字]倍"
- **产品视角**：不只是技术分析，要回答"这个东西能做成什么产品？谁会用？"
- **GitHub 链接必带**：X/B站放正文，小红书/抖音放评论区

### X Thread 模板

```
1/n: Just found a GitHub repo that [一句话价值].
     [GitHub链接]

2/n: What it does:
     - [亮点1 + 具体数字]
     - [亮点2 + 具体数字]
     - [亮点3 + 具体数字]

3/n: Why it matters:
     [产品视角洞察 1-2 句]
     This isn't just a tool. It's a [更大趋势] signal.

4/n: Who should care:
     - [人群1]
     - [人群2]
     Star it: [GitHub链接]
```

## 3. 热点事件讲解

> **本质**：不是"新闻搬运"，是"帮用户看懂热点背后的逻辑"。速度 > 深度（但深度是 IP 护城河）。

### 结构模板

```
[事件一句话] → 关键信息 3 条 → 深度解读（为什么重要）→ 我的判断 → CTA
```

### 平台适配

| 平台 | 形式 | 结构 | 钩子 | 时效要求 |
|------|------|------|------|---------|
| 小红书 | 快速图文 1-3 张 | 封面：事件关键词+一句话判断 / 正文：3要点+解读 | 反认知 | 6h内 |
| X | 独狼帖 | `[事件] + [1句判断] + [为什么]` | 反认知 | 2h内 |
| 抖音 | 口播 | 3秒（事件钩子）→ 3个要点 → 1句判断 | 痛点共鸣 | 6h内 |
| B站 | 深度分析 | 开场 → 事件回顾 → 深度解读 → 行业影响 → 个人观点 | 反认知 | 24h内 |

### 关键原则

- **速度**：热点 6h 内发 = 流量红利，24h 后发 = 深度补充
- **判断优先**：用户不缺信息，缺判断。先给结论，再给依据
- **个人观点**：热点讲解的灵魂是"Ryder 怎么看"，不是"客观报道"
- **小红书/X 发快讯，B站发深度**：同一热点可以2次创作

### 快手标题党专用

> 快手已移除（用户画像不匹配）。如需标题党风格，仅在抖音适度使用。

- `太炸裂了！[事件]背后的真相`
- `打工人睡不着了！[事件]意味着什么`
- `看完[事件]我破防了，AI时代真来了`
- `[事件]刷屏了，但99%的人没看懂`

## 4. 日常感悟

> **本质**：不是"鸡汤"，是"用个人故事建立 IP 人设"。让用户觉得"Ryder 是个有趣的人"。

### 结构模板

```
[具体场景] → 我的第一反应 → 反思后的认知 → 一句话升华 → 软性 CTA
```

### 平台适配

| 平台 | 形式 | 结构 | 钩子 | 字数/时长 |
|------|------|------|------|----------|
| 小红书 | 图文 1 张 | 封面：金句 / 正文：场景→反思→升华 | 生活方式 | ≤200字 |
| X | One-liner | `[1句场景] + [1句反转认知]` | 生活方式 | ≤280字符 |
| 抖音 | 口播 | 3秒（场景钩子）→ 故事 → 1句升华 | 痛点共鸣 | 20-30s |
| B站 | — | 不推荐（B站不适合短感悟，除非做成系列vlog） | — | — |

### 关键原则

- **具体 > 抽象**：不说"最近很忙"，说"这周开了14个会，写了3份PRD"
- **反共识 > 共识**：不说"AI很重要"（所有人都知道），说"我用了3个月AI后发现，最该自动化的不是工作，是开会"
- **留白 > 说透**：感悟不是说教，留空间让用户自己悟
- **频率**：日常感悟是 IP 的"温度感"，不是主菜。1周1条够了

### X One-liner 模板

```
[具体场景，带数字].
[反转/反共识，1句].
```

例：
```
Spent 3 hours debugging a prompt.
Turns out the bug was my assumption, not the model.
```

## 跨平台内容改编原则

**1 份内容 → 多平台改编，不是复制粘贴**：

| 维度 | 小红书 | X | 抖音 | B站 |
|------|--------|---|------|-----|
| 语气 | 温暖共鸣 | 硬核极简 | 口语直接 | 理性深度 |
| 长度 | 200-300字 | 280字符 | 30-60s | 5-15min |
| 钩子 | 痛点/身份 | 反认知/对比 | 痛点/对比 | 圈层/反认知 |
| 视觉 | 图文卡片 | 文字+可选图 | 录屏/口播 | 分镜长视频 |
| CTA | 软性（评论/收藏） | 链接 | 关注/评论区 | 三连+关注 |

**改编工作流**：
1. 写**核心信息**（1句话：这件事的价值是什么？）
2. 选**主钩子**（7大钩子选1）
3. 按平台**改写结构和语气**（不是翻译，是重构）
4. 各平台**独立验证**（不假设同一套话术通用）

## 内容类型 × IP 矩阵占比

| 内容类型 | 建议占比 | IP 作用 | 频率 |
|---------|---------|---------|------|
| AI产品推荐 | 30% | 专业感 + 工具人设 | 2-3条/周 |
| GitHub深度分析 | 25% | 技术深度 + 圈层认同 | 1-2条/周 |
| 热点事件讲解 | 20% | 时效性 + 判断力 | 蹭热点，不强制 |
| 日常感悟 | 15% | 温度感 + 人格化 | 1条/周 |
| 互动/福利 | 10% | 粉丝粘性 | 1条/2周 |

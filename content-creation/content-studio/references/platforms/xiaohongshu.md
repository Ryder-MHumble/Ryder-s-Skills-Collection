# 小红书（Xiaohongshu）平台方法论

> **核心**：心理钩子 + 极简 + 强共鸣。**标题和封面是命门**。
> 用户在小红书是"刷"的状态，3 秒定生死。

## 1. 平台调性速查

| 维度 | 数值 |
|------|------|
| 标题 | ≤ **20 字**（超长被截） |
| 正文 | **200-300 字**（短 = 完读率高） |
| 封面文字 | ≤ **12 字** |
| 图片数量 | 1（封面）/ 3-9（图集） |
| 钩子位置 | 标题 + 第 1 段前 2 句 |
| 视觉资产 | **3:4 封面**（1080×1440 必选） |
| 主导心理钩子 | 痛点共鸣 / 群体认同 / 身份贩卖 |

## 1.1 Deep Research 图集特例（证据链内容）

当内容来自 deep research / social-critique / 调研报告 / 学术报告时，小红书仍要保持强共鸣，但图集不能过空。

**强制要求**：
- 图集最后一页必须是 `References / 参考材料`。
- References 页不放 URL，只列资料名称，格式为 `[1] 资料名称`。
- 内容卡中所有数据、案例、事实判断都要带参考材料号，如 `[1]`、`[2][4]`。
- 一张内容卡至少承载 2-3 个信息点，不做“只有一句大字”的空卡。

**推荐卡片密度**：
- 封面：≤ 12 字主标题 + 1 句副标题。
- 内容卡：1 个主判断 + 2-3 个支撑点，或 1 个数据 + 1 个解释 + 1 个影响。
- 正文长度：每张内容卡 90-160 个汉字；重点卡最多 180 字，但必须分块。
- 引用编号：放在数据、案例、引用句或页脚，不抢主视觉但必须清楚。

**图集结构**：
```
01 封面：反认知钩子
02 核心判断：一句话结论 + 2 个原因 [1][2]
03 数据卡：一个硬数据 + 解释 + 影响 [1]
04 机制卡：为什么会这样 + 结果 [2][3]
05 案例卡：劳动者/用户声音 + 提炼 [4]
06 立场边界：不反技术，反分配机制 [2][3]
07 References：列 [1]-[6] 资料名称，不放 URL
```

## 1.2 图集发布文案：图片主载体，正文辅助

当小红书内容是「图集」时，正文不要复述整组图，也不要写成报告摘要。图片负责讲清楚事实，正文负责让用户愿意点开、愿意滑、愿意评论。

**标题不是套公式，是抓住体感**：
- 不要固化「反问 + 情绪判断」这类结构；那会变成模板味。
- 先提炼用户给的标题为什么吸引人：它通常有口语感、反差、被戳中的体感、轻微情绪张力。
- 每个标题候选都要像真实用户会说的话，而不是运营写出来的栏目名。
- 同一组标题要换情绪角度，不要换几个词复用同一骨架。

**标题风格检查**：
| 维度 | 合格表现 | 不合格表现 |
|------|----------|------------|
| 停顿感 | 刷到会停一下 | 像论文题目 |
| 共鸣感 | 直接戳中用户日常感受 | 只描述宏观现象 |
| 反差感 | 承诺与现实有冲突 | 没有张力 |
| 口语感 | 像人话，有轻微吐槽 | 像宣传语 |
| 兑现感 | 图集内容能支撑标题 | 为点击夸大 |

**正文不是图集摘要，是导流文案**：
```
开头：先写一个具体体感或工作瞬间。
中段：写 2-3 个具体动作/对话，让人对号入座。
转折：点出核心矛盾，但证据留给图片。
结尾：问一个真实问题，让用户愿意讲自己的经历。
```

**风格拆解，不保存固定句子**：
- 开头要像一个真实体感突然冒出来，而不是报告导语。
- 第二段要写具体动作链，让用户立刻代入自己的工作流。
- 转折要把“工具提效”和“人更累”的矛盾点破，但把证据留给图片。
- 结尾问真实处境，不问泛泛的“你怎么看”。

**反模式**：
- ❌ “报告显示……”开头，像摘要。
- ❌ 正文里塞满资料名和引用编号，影响阅读。
- ❌ 把图里每一页重新讲一遍。
- ❌ 标题太平，如“AI与劳动效率研究”。
- ❌ 记住几个爆款标题结构后机械换词。

## 2. 7 大钩子在小红书的应用

| 钩子 | 公式 | 标题例子 |
|------|------|----------|
| **痛点共鸣** | 痛点场景 + 扎心金句 + 对号入座 + 软性 CTA | 「打工人最大的恐惧：你救了一辈子火，老板以为你只会回'收到'」 |
| **身份贩卖** | 高段位身份 + 暗示行为 + 揭晓 + 软性 CTA | 「高段位打工人，都藏着一个 🟠 系统思考者」 |
| **反认知震撼** | 我以为 X + 都是错的 + 新认知 + 软性 CTA | 「我打了 3 年工才知道：我不是 INTJ，是 🟠」 |
| **群体认同** | 身份标签 + 共同特征 + 对号入座清单 + 软性 CTA | 「打工人集合：你中了 3 条以上，就是 🟠」 |
| **对比冲击** | 对比项 1 vs 2 + 结果 + 用户选择 | 「测了 16 次 MBTI 不如让 AI 读 1 年钉钉」 |
| **圈层暗号** | 圈层黑话 + 暗示 + 揭晓 + 圈层归属 | 「懂的人自然懂：1.2 万条'收到'里藏着的 🟠」 |
| **生活方式** | 生活方式 + 具体场景 + 成为方式 + 软性 CTA | 「想成为'看穿一切'的打工人？」 |

## 3. 极简三段式（小红书标准结构）

```
段 1（1-2 句）：心理钩子（痛点 / 身份 / 反认知）
段 2（2-3 句）：对号入座清单（3-4 条具体场景）
段 3（1 句）：软性 CTA（让用户自己行动）
```

## 4. 标题风格钩子（只学味道，不套结构）

这些不是标题公式。使用时只提炼“为什么吸引人”，不要机械照抄骨架。

| 钩子 | 学习的味道 | 适合场景 |
|---|---|---|
| 反常识 | 先戳破一个大家默认相信的承诺 | AI减负、效率神话、工具崇拜 |
| 群体共鸣 | 让用户觉得“这说的是我们这类人” | 打工人、产品经理、程序员、学生 |
| 具体痛感 | 写一个真实工作瞬间，而不是抽象概念 | 加班、返工、开会、被催交付 |
| 趋势反差 | 一边变强，另一边变糟 | 工具更强但人更累、效率更高但自由更少 |
| 红利追问 | 把问题从“有没有用”转向“归谁” | 劳动、商业、组织管理 |
| 轻微吐槽 | 有一点不爽，但不骂街 | 职场观察、社会现象批判 |
| 自我怀疑 | 先承认自己的体感，再用图集支撑 | “我越来越觉得…”、“我开始怀疑…” |
| 冷静点破 | 不煽情，但一句话把机制说透 | 深度调研、证据链图集 |

**生成标题时的要求**：
- 每次至少给 5 个候选，但候选之间必须是不同情绪角度，不是同一结构换词。
- 如果用户给了好标题，只学习其语感与吸引力来源，不把它保存成固定模板。
- 标题可以有锋芒，但图片必须能兑现标题承诺。

## 5. 强 CTA 公式（让用户行动）

- ✅ "评论猜你是 🟠🟢🟣🔵 哪个"
- ✅ "🔥 点赞过 500 出下条对照表"
- ✅ "评论区告诉我你中了几个"
- ❌ "评论告诉我"（太弱）
- ❌ "快来试试"（强推）

## 6. 封面设计要点（视觉核心）

| 元素 | 规范 |
|------|------|
| 比例 | **3:4**（1080×1440） |
| 主标题 | ≤ 12 字，大字号，**前 3 秒可读** |
| 副标题 | 1 行小字（可选） |
| 配色 | 与品牌色一致，**1-2 种主色** |
| 字体 | 1 种（推荐：思源黑体 / Inter / Plus Jakarta Sans） |
| 元素 | 1 个视觉锚点（emoji / 符号 / 数字） |
| 留白 | 充足（>30% 空白） |

**推荐视觉风格**：
- 知识类 / 工具类 → **McKinsey Business**（克制专业）
- 个人向 / 生活向 → **Notion Minimal**（温暖极简）
- 圈层 / 痛点 → **Neo-Brutalism**（冲击感）
- 视觉化强 → **Swiss Grid**（结构感）

## 7. 标签策略（影响分发）

- **3-5 个标签**（不要堆）
- 1 个大标签（百万级曝光，如 #AI）
- 2 个中标签（万级，如 #AI工具）
- 1-2 个小标签（精准，如 #AI产品经理）

## 8. 反模式

- ❌ 标题 = 产品名
- ❌ 正文分点列表（用叙事）
- ❌ emoji 矩阵（刷屏感）
- ❌ 长篇大论（>500 字完读率暴跌）
- ❌ "首先...其次..."
- ❌ 自我介绍
- ❌ 强推产品
- ❌ 把所有原型都罗列
- ❌ 写"怎么用"（不是产品文档）
- ❌ 形容词（"创新的"、"智能的"）
- ❌ 标题 > 20 字（被截）
- ❌ 封面文字 > 12 字（看不清）

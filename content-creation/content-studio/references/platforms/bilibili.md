# B站（Bilibili）平台方法论

> **核心**：深度 + 逻辑 + 弹幕互动。**B站用户愿意看长视频，但要求信息密度和逻辑性**。
> B站是"看"的状态，不是"刷"——用户主动搜索、主动追更，**内容质量 > 钩子技巧**。

## 1. 平台调性速查

| 维度 | 数值 |
|------|------|
| 视频时长 | **5-15min** 为主，深度内容 20-30min |
| 脚本字数 | **1500-4000 字**（口播 1 字 ≈ 0.3s） |
| 钩子位置 | **前 15 秒** + 标题 + 封面 |
| 封面文字 | ≤ **12 字**（大字号，信息明确） |
| 视觉资产 | **16:9 横屏**（1920×1080 必选） |
| 字幕 | **全程字幕**（B站用户对字幕要求高） |
| 主导心理钩子 | 反认知 / 圈层暗号 / 对比冲击 |
| 调性 | **硬核、深度、逻辑清晰、知识感** |
| 主导分区 | 知识区 / 科技区 / 数码区 |

## 2. 内容结构（开场白 + 逻辑展开 + 深度分析 + 互动结尾）

```
0-15s   ：开场白 hook（悬念 / 反认知 / 问题抛出）
15-60s  ：背景铺垫（为什么讲这个 + 你的视角）
1-5min  ：逻辑展开（分点论述，每点有论据）
5-10min ：深度分析（独家观点 / 代码演示 / 对比实验）
10-13min：总结升华（金句 + 价值回顾）
13-15min：互动结尾（弹幕引导 + 三连 + 下期预告）
```

**关键**：B站用户能接受长视频，但**每 2 分钟必须有信息增量**，否则弹幕刷"水视频"。

## 3. 4 种内容类型适配

| 内容类型 | 时长建议 | 钩子策略 | 结构要点 |
|----------|----------|----------|----------|
| **AI 产品推荐** | 8-12min | "测了 20 个 AI 工具，只有这 3 个我天天用" | 痛点 → 测评方法 → 逐个深度演示 → 适用人群 → 总结推荐 |
| **GitHub 分析** | 10-15min | "GitHub 3 万星项目，源码我扒了" | 项目背景 → 架构分析 → 核心代码解读 → 实操演示 → 上手建议 |
| **热点讲解** | 5-10min | "AI 圈今天的大事件，3 个层面分析影响" | 事件还原 → 关键信息 → 多角度分析 → 对普通人的影响 → 你的预判 |
| **日常感悟** | 5-8min | "做了 3 年 AI 产品经理，我踩过的 5 个坑" | 反认知开头 → 故事展开 → 复盘分析 → 经验提炼 → 共鸣金句 |

## 4. 长视频脚本模板（分镜 + 口播 + 画面说明）

```
【开场白 hook】（0-15s）
口播：[悬念/反认知/问题抛出，1-2 句]
画面：[吸引眼球的视觉 / 关键数据 / 悬念画面]
字幕：[核心信息高亮]

【背景铺垫】（15-60s）
口播：[为什么讲这个 + 你的独特视角]
画面：[个人出镜 / 相关素材混剪]
字幕：[关键词强调]

【正文 Part 1】（1-3min）
口播：[第一个核心论点 + 论据]
画面：[PPT / 代码截图 / 产品演示 / 流程图]
字幕：[分点标题 + 关键数据]

【正文 Part 2】（3-6min）
口播：[第二个论点 + 深度展开]
画面：[对比实验 / 实操录屏 / 架构图]
字幕：[对比表格 / 关键结论]

【正文 Part 3】（6-10min）
口播：[第三个论点 / 独家观点]
画面：[独家素材 / 源码解读 / 深度演示]
字幕：[金句高亮]

【总结升华】（10-13min）
口播：[价值回顾 + 升华金句]
画面：[总结卡片 / 思维导图]
字幕：[核心结论]

【互动结尾】（13-15min）
口播：[弹幕互动 + 三连引导 + 下期预告]
画面：[出镜 / 下期预告片]
字幕：[互动问题 + "一键三连"]
```

### 脚本实例（GitHub 深度分析）

```
[0-15s] 口播：GitHub 上一个 3 万星的项目，我扒了源码，发现它比你想的强太多。
        画面：项目首页 + star 数飙升曲线
        字幕：「3 万星 · 源码解读」

[15-60s] 口播：这个项目叫 XXX，做的是 XXX。今天我从架构、核心代码、实操三个层面给你讲清楚。
        画面：个人出镜 + 项目 logo
        字幕：架构 / 代码 / 实操

[1-5min] 口播：先说架构。它用了 XXX 模式，核心是 XXX...
        画面：架构图动画 + 代码高亮
        字幕：分点标注关键模块

[5-10min] 口播：再看核心代码。这段是整个项目的灵魂...
        画面：源码逐行解读 + 终端演示
        字幕：关键代码行高亮

[10-13min] 口播：总结一下，这个项目适合 XXX 的人，如果你是 XXX 就别用了。
        画面：总结思维导图
        字幕：适用人群 / 不适用人群

[13-15min] 口播：你觉得这个项目怎么样？弹幕告诉我。下期我讲 XXX，点关注不迷路。
        画面：出镜 + 下期预告
        字幕：「弹幕互动」「一键三连」
```

## 5. 标题规范（数字 + 关键词 + 悬念）

| 公式 | 例子 |
|------|------|
| **数字 + 关键词 + 悬念** | 「3 万星 GitHub 项目源码解读，比你想的强太多」 |
| **反认知 + 主体** | 「测了 20 个 AI 工具，只有 3 个我天天用」 |
| **数字 + 对比 + 结论** | 「5 款 AI 周报工具横评，第 1 名我没想到」 |
| **热点 + 深度 + 视角** | 「AI 圈今天炸了，3 个层面深度分析」 |
| **身份 + 经验 + 数字** | 「3 年 AI 产品经理，踩过的 5 个坑」 |
| **问题 + 悬念 + 价值** | 「AI 真的能取代产品经理吗？我做了个实验」 |

**标题要点**：
- ≤ **30 字**（B站标题空间充足）
- 必含 **1 个数字**（具体感）
- 必含 **1 个关键词**（SEO + 搜索匹配）
- 可含 **1 个悬念词**（"没想到""比你想的""真相"）
- **不要标题党**（B站用户反感，会刷"标题党取关"）

## 6. 弹幕互动设计

B站特色：**弹幕是内容的一部分**，好的视频会"引导弹幕"。

| 时机 | 弹幕引导方式 | 例子 |
|------|-------------|------|
| 开场 15s | 抛出问题，引导弹幕回答 | "你用过几个？弹幕打数字" |
| 关键转折 | 预埋"名场面"，引导刷屏 | "前方高能" / 故意停顿 |
| 观点输出 | 留出争议空间，引导讨论 | "你同意吗？弹幕告诉我" |
| 数据展示 | 引导弹幕刷反应 | "看到这个数据我直接好家伙" |
| 结尾 | 互动问题 + 三连引导 | "下期讲什么？弹幕投票" |

**弹幕预埋技巧**：
- 在关键信息处**故意停顿 1-2 秒**，给用户发弹幕的时间
- 在反认知观点后**加一句"你品，你细品"**，引导弹幕刷"确实"
- 在数据对比处**放大数字 + 音效**，引导弹幕刷"好家伙""离谱"
- **不要强行要弹幕**（"快发弹幕"——用户反感）

## 7. 封面设计要点

| 元素 | 规范 |
|------|------|
| 比例 | **16:9**（1920×1080） |
| 主标题 | ≤ 12 字，大字号，**信息明确** |
| 视觉主体 | 1 个核心视觉（人物 / 产品 / 数据图） |
| 配色 | 高对比，**避免花哨** |
| 字体 | 思源黑体 / 阿里巴巴普惠体（粗体） |
| 风格 | 知识感 > 营销感（B站用户反感营销封面） |

**推荐视觉风格**：
- 知识区 / 科技区 → **Swiss Grid**（结构感、专业）
- 深度分析 → **McKinsey Business**（克制、权威）
- 热点讲解 → **Neo-Brutalism**（冲击感，慎用）
- 个人向 / 感悟 → **Notion Minimal**（温暖、真诚）

## 8. 反模式

- ❌ 开头自我介绍超过 15s（B站用户也没耐心）
- ❌ 信息密度低（"水视频"弹幕警告）
- ❌ 标题党（B站用户最反感，会刷"取关"）
- ❌ 封面营销感太强（像广告 → 不点）
- ❌ 无字幕（B站用户对字幕要求高）
- ❌ 口播无逻辑（B站用户爱挑逻辑漏洞）
- ❌ 强行要三连（"求三连"——用户反感）
- ❌ 视频时长 < 5min（B站算法不友好）
- ❌ 横竖屏混用（必须 16:9 横屏）
- ❌ 照搬抖音/快手风格（B站文化完全不同）
- ❌ 不做分镜规划（长视频必须分镜）

## 9. 发布时间建议

| 时段 | 效果 |
|------|------|
| 工作日 18:00-20:00 | 下班后观看高峰 |
| 工作日 21:00-23:00 | **晚间黄金时段**（最强） |
| 周末 10:00-12:00 | 周末早高峰 |
| 周末 14:00-17:00 | 周末下午（长视频观看高峰） |
| 周末 20:00-23:00 | 周末晚高峰 |

**避免**：工作日 9:00-17:00（B站用户以上班族/学生为主，白天数据差）

**发布频率**：B站长视频制作周期长，**周更 1-2 条**即可，质量 > 数量。

## 10. KPI 速查

| 指标 | 健康值 | 说明 |
|------|--------|------|
| 完播率 | > **20%** | 长视频完播率天然低，20% 已优秀 |
| 5 分钟留存 | > **40%** | 前 5 分钟是关键 |
| 三连率 | > **5%** | B站核心指标（点赞+投币+收藏） |
| 弹幕率 | > **3%** | B站特色，互动深度 |
| 关注转化 | > **2%** | B站粉丝粘性最高 |

## 11. B站专属：系列化运营要点

- **系列化 > 单条爆款**：B站用户爱追更，"AI 工具测评系列"比单条更有价值
- **分区选择**：AI 内容优先投「科技区-数码」或「知识区-科技」
- **标签策略**：5-10 个标签，1 个大标签（#AI）+ 精准长尾标签
- **简介区**：放时间轴（00:00 开场 / 01:30 架构分析...），B站用户爱跳看
- **置顶评论**：自己发一条置顶评论，补充信息或引导互动
- **动态预告**：发布前 1 天发动态预热，B站用户会蹲更

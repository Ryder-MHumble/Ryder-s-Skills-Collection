# X (Twitter) 文案模板

> 使用方法：复制此模板，填入具体内容

## 元信息

```yaml
project: {项目名}
sub_style: {6 选 1} # indie hacker / terminal / tech declaration / hot take / OS announcement / one-liner
hook: {7 大钩子选 1}
style: {心理钩子 / 营销号 listicle / 混合}
lang: {en / mixed} # X 优先英文
```

## sub-style 决策

```markdown
- CLI / 命令行工具 → Terminal Output
- 性能 / 架构亮点 → Tech Declaration
- 个人项目 / side project → Indie Hacker
- 项目 launch → OS Announcement
- 反共识 → Hot Take
- 视觉化 → One-liner
- 拿不准 → Indie Hacker（最安全）
```

## 独狼帖（≤ 280 字符）

```markdown
[钩子 + 1 段 + 1 链接]
```

### 模板 A：Indie Hacker

```
Built [工具] that [能力 1], [能力 2], and [能力 3].

No [烂大街 1]. No [烂大街 2].

Just [核心价值].

github.com/...
```

### 模板 B：Terminal Output

```
$ [tool] [args]
Loading [数字]...
Output: [揭晓]

Reason: "[1 句]"

Open source. github.com/...
```

### 模板 C：Tech Declaration

```
[1 句总述]

- [metric]: [value]
- [metric]: [value]
- [metric]: [value]
- [metric]: [value]

github.com/...
```

### 模板 D：Hot Take

```
Hot take: [争议观点].

Built something better.

[1 行方案] + [特点] + 链接
```

## Thread（3-4 条）

### 1/n：钩子 + 反转

```
[心理钩子关键词: I thought / The one who / Want to] + [反转]
```

### 2/n：数字证据

```
[3-5 个 bullet]
- 数字 1
- 数字 2
- 数字 3
```

### 3/n：对号入座

```
[3-4 条具体场景]
- 场景 1
- 场景 2
- 场景 3
- 场景 4
```

### 4/n：开放问题 + 软性 CTA

```
[开放问题] + [软性 CTA，不推产品]
```

## 反模式自检

- [ ] 不自我介绍
- [ ] 不"首先...其次..."
- [ ] 不强推
- [ ] 不 emoji 开头
- [ ] Thread ≤ 7 条
- [ ] 不呼吁关注
- [ ] 不 tag 堆
- [ ] 不写"怎么用"
- [ ] **不把中文钩子直译成英文**

## 视觉资产

- 首图：`cover.png`（16:9 = 1600×900）
- Thread 卡片：`cards/card-XX.png`（16:9）
- 风格：tech → Swiss Grid / hot take → Neo-Brutalism

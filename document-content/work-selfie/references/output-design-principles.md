# self-distill 输出设计原则

> 来源：2026-06-11 v1 重构会话。三条核心原则从用户反馈中沉淀：
> 「未分类选手空模块」「流程错了」「信息密度不够」。

## 原则 1 · 先报告后卡片（架构层）

**报告 = source of truth，卡片 = 报告的可视化摘要。**

```
采集 → 分析 → 完整报告（10 章深度）→ 渲染卡片（基于报告）→ 本地保存 → [可选] 发钉钉
                                          ↑
                                卡片从 analysis dict 派生，不重新分析
```

**为什么**：
- 用户原话：「深度分析完用户全部聊天记录后你需要生成一般完整的深度分析结果报告以消息的形式发送给用户，然后再根据这个完整的分析报告生成渲染在人物卡片上的信息报告模版，里面的内容必须要足够多且足够清晰」
- 卡片是「摘要」不是「替代品」——如果跳过报告直接渲染卡片，会丢失 80% 的分析深度
- 报告是用户最终要读的内容（消息、文档、Markdown），卡片只是社交分享载体

**反模式**：
- ❌ 直接从 `analysis` 拼卡片，跳过 `build_report_text`
- ❌ 卡片包含报告里没有的字段（信息来源不一致）
- ❌ 报告太短（<500 字），卡片再丰富也是空中楼阁

**实施**：见 `scripts/main.py` Step 4a-4b 的拆分。

---

## 原则 2 · 空字段兜底（绝不出"未分类"）

**任何推断字段为空时返回有意义的人设标签 + 一句话总结，禁用「未分类」「????」「数据不足」**

```python
# ❌ 禁用
{"label": "未分类选手", "code": "????", "desc": "数据样本不够"}

# ✅ 推荐
{"label": "均衡型", "code": "均衡", "desc": "暂无足够数据画像，但既然你点开了，说明你对自己有好奇"}
```

**适用范围**：
- SBTI 推断（8 候选 Top1）空 → "均衡型"
- 5 维推断（决策/沟通/压力/价值观/信息处理）每维都返回 primary + secondary（永远不空 secondary）
- 动物画像空 → "变色龙"（本来就是兜底）
- 协作 Top 3 空 → 显示「无显著协作对象」而非空数组

**5 维推断的 secondary 必须永远存在**：

```python
def infer_decision_style(e, b):
    scored = [...]  # 4 个候选打分
    scored.sort(key=lambda x: -x["score"])
    primary = scored[0]
    secondary = scored[1] if len(scored) > 1 else None  # 永远取第二名
    return {"primary": ..., "primary_score": ..., "secondary": ..., "secondary_score": ..., "all": scored}
```

**反模式**：
- ❌ 卡片 5 维推断条上有「?」或空白格
- ❌ 报告里出现「数据样本为零，按 50 分兜底」这种技术话术
- ❌ 顶层 report 章节用「未分类」「N/A」当标题

**用户原话**：「中间还有个未分类选手的信息模块是空的，说明分析的有问题，这里应该放类似一句话总结比如「内卷选手」然后加一句话总结」

## 原则 3 · 卡片信息密度（默认 4:5 删除协作模块）

**卡片不是头像，是「人物档案速览」——信息密度要高，多模块铺满。**

当前 4:5 模板默认 720x900，右侧只保留 3 个数据模块 + 底部 slogan；协作榜留在完整报告，不进 PNG 卡片（信息源全部从 `analysis` 派生）：

| # | 模块 | 数据源 | 展示形式 |
|---|------|--------|---------|
| 01 | 职场 SBTI | `sbti_top3[0]` | 左侧小信息卡 + 新野兽派方块 |
| 02 | **SBTI 8 候选排行** | `sbti_top3` + 兜底 | 8 条横向条形 |
| 03 | **5 维人格** | `decision_style` 等 5 个 | 5 行 primary + secondary |
| 04 | **表达 DNA** | `expression_dna` | 4 行（Top 5 词/长度/工作词比/情绪） |
| 底部 | 职场宣言 | `slogan_long` | 黑底黄字承接底部空间 |

**新模块必须配 helper function**（见 `render_card.py` 末尾）：

```python
def build_sbti_8_html(sbti_top3): ...
def build_traits_5d_html(analysis): ...
def build_expression_dna_html(expression_dna): ...
```

**反模式**：
- ❌ 5 维推断条只显示 primary 一行（浪费空间）
- ❌ SBTI 只显示 Top1（用户希望看到 8 候选横向对比）
- ❌ 表达 DNA 只显示长度均值（关键词 + 情绪 + 工作词比 都要）
- ❌ 默认 4:5 PNG 再放协作 Top 5（报告里可以保留，卡片里不要）

---

## 三大原则的协作

| 场景 | 原则 1（架构） | 原则 2（兜底） | 原则 3（密度） |
|------|--------------|--------------|--------------|
| 数据充足 | 报告 10 章 + 卡片 5 模块 | 真实推断 | 全模块铺满 |
| 数据稀少 | 报告章节可省（条件渲染） | "均衡型" + 解释 | 仍铺 5 模块（只是内容变兜底） |
| 首次运行 | 报告 + 卡片都生成 | 同上 | 同上 |
| 增量运行 | 报告多 1 章「历史对比」 | 同上 | 同上 |

**核心**：三大原则是 AND 关系，不是 OR。任何场景下都要全部满足。

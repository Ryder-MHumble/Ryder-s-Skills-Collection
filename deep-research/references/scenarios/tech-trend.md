# 场景模版：tech-trend（技术趋势追踪）

## 适用 query 类型

- 「过去 N 天/月的 XX 技术演进」「XX 框架最新论文」「XX 路线图」
- 「XX 方向有什么新突破」「XX 综述」
- 关键词：趋势/演进/论文/框架/路线图/综述/技术路线/突破/SOTA

## 报告骨架（严格 5 章）

```
## 1. 一句话技术判断
## 2. 关键技术节点时间线
## 3. 三大代表性工作深度解析
## 4. 主要技术路线 / 阵营分析
## 5. 下一步观察清单 + 不确定项
```

**章节数：5 个 ##，章节内不嵌套 ###**

## 章节内容指引

### 1. 一句话技术判断

- 一句话给出现状（30 字内）
- 给出窗口期内最重要的 1 个判断（50 字内）
- 格式：**当前现状** + **核心判断** + **窗口期（YYYY-MM-DD ~ YYYY-MM-DD）**

### 2. 关键技术节点时间线

- 严格按时间顺序列出窗口期内的关键技术节点
- 每个节点用日期 + 一句话描述 + L1/L2/L3 信源标注
- 不少于 5 个节点，不少于 8 个节点（视数据丰富度）
- 格式示例：
  ```markdown
  - **2026-04-15**：[L1, Anthropic 博客] Claude 3.7 Sonnet 发布，首发 Computer Use 能力。
  - **2026-04-22**：[L2, arXiv:2504.12345] 提出 ScreenAgent 多模态 GUI 框架。
  - **2026-05-03**：[L3, 36氪] 国内某厂商上线 GUI Agent 内测。
  ```

### 3. 三大代表性工作深度解析

- 选 3 个最重要的代表性工作（论文/产品发布/开源项目），按影响力排序
- 每个工作包含：标题（中英对照）/ 作者或团队 / 发布时间 / arXiv ID 或 URL / 核心贡献（一段）/ 局限性（一段）
- 论文类工作必含 arXiv ID 和 PDF 链接
- 格式：**工作标题**（English, arXiv:XXXX.XXXXX） - 核心贡献段落 - 局限性段落

### 4. 主要技术路线 / 阵营分析

- 识别当前主流技术路线（2-4 条）
- 每条路线包含：代表团队 / 核心思想 / 优势 / 劣势 / 代表工作
- 用「加粗引领句 + 段落」格式，不用表格
- 格式示例：
  ```markdown
  **路线 1：端到端多模态大模型路线** - 代表团队 OpenAI/Anthropic。核心思想是将屏幕截图直接输入多模态模型，模型输出操作指令。优势是泛化性强，劣势是推理成本高。代表工作：GPT-4V Operator、Claude 3.7 Computer Use。

  **路线 2：模块化 Pipeline 路线** - 代表团队 Adept/Airkit。核心思想是分解为视觉理解+动作规划+执行三模块。优势是可解释性，劣势是模块间误差累积。代表工作：Adept ACT、阿里 Airkit。
  ```

### 5. 下一步观察清单 + 不确定项

- 下一步观察清单：3-5 个具体可验证的预测（如「2026-Q3 之前 Anthropic 是否会开源 Computer Use 框架」）
- 不确定项：2-4 个目前数据不足无法判断的点
- 格式：
  ```markdown
  **下一步观察**：
  1. 预测 A — 验证方式：...
  2. 预测 B — 验证方式：...

  **不确定项**：
  - 某厂商的能力边界（仅 L3 单源数据）[L3 单源, 2026-04-15]
  ```

## 信源调度

### L1 必调（核心技术领导方）
- 各主要技术领导方官方博客（Anthropic、OpenAI、Google DeepMind、Meta AI、HuggingFace、阿里、字节、深度求索等）
- 各公司 GitHub Release（如果 query 涉及框架/工具）

### L2 必调（核心学术信源）
- arXiv：搜 `cat:cs.AI`、`cat:cs.CL`、`cat:cs.CV` + query 关键词，时间窗口筛选
- OpenAlex：搜 query 关键词 + 高引工作 + 近 window 内
- 顶会论文（NeurIPS/ICML/ICLR/CVPR/ACL 等）：通过 OpenAlex 过滤 venue

### L3 必调（行业媒体交叉验证）
- 中文：36氪/晚点/虎嗅 搜 query 关键词 + 时间窗口
- 英文：TechCrunch/The Verge 搜 query 关键词

### L4 可选
- kol-interview-to-wiki 的 Obsidian 目录里搜 KOL 名字 + query 关键词
- 如 KOL 已有深度观点引用，强相关时纳入

### L5 可选（社区讨论，仅作为补充）
- X 搜 query 关键词 + `filter:verified` + 时间窗口
- HN Algolia 搜 query 关键词

## 强约束自检清单

- [ ] 章节数 = 5（不嵌套 ###）
- [ ] 时间线节点 ≥ 5 个，按时间顺序
- [ ] 三大代表性工作必须含 arXiv ID 或官方 URL
- [ ] 至少 3 个信源层级被使用（L1+L2+L3 必到）
- [ ] 信源标注完整（[L?, 日期]）
- [ ] 中英对照齐全
- [ ] 不确定项单独列出，不藏在正文里

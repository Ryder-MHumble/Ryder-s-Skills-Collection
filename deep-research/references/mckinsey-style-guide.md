# 麦肯锡风样式指南（v1.3 新增，可选 Word 输出专用）

> 本指南用于 `scripts/word_output.py`（麦肯锡风 .docx 渲染器）和其他需要"咨询风"视觉的场景。**Markdown 默认输出不受此约束**——只对 `--word` / `--docx` 标志触发的 Word 输出生效。

## 1. 配色系统（强制 2 色 + 中性灰）

| 用途 | 颜色 | HEX | 使用场景 |
|------|------|-----|----------|
| 主色 | 深蓝 | `#1F4E79` | 标题、表头、超链接、章节封面 |
| 辅色 | 中性灰 | `#7F7F7F` | 次要文字、副标题、注释 |
| 浅灰背景 | 浅灰 | `#F2F2F2` | 表格交替行底色 |
| 强调色（仅关键数字）| 深红 | `#C00000` | 关键数字粗体大字、错误警示 |
| 纯白 | 白 | `#FFFFFF` | 背景、表格白行 |

**禁用配色**：
- ❌ 5+ 色花里胡哨（粉/蓝/绿/黄/红多色 callout 块）
- ❌ 浅红/浅绿/浅黄色 callout 背景（看起来花里胡哨）

## 2. 字体设置

- **中文**：SimSun（宋体）
- **英文/数字**：sans-serif
- **正文**：10.5pt
- **二级标题**：14pt 加粗
- **一级标题**：18-22pt 加粗
- **关键数字**：16-18pt 加粗（深蓝或黑色，不用红色——红色只用于警示）
- **注释/来源**：8.5pt 中性灰

**行距**：1.4-1.6
**段后间距**：4-6pt
**章节前间距**：8-12pt

## 3. 章节封面（克制的麦肯锡风）

```
─────────────────────────────────────
  第 N 章
  章节标题（大字 22pt 深蓝加粗）
  英文副标题（小灰 11pt 斜体）
─────────────────────────────────────
```

**不**用：大块色块背景 / 大图标 / 大 emoj 装饰。

## 4. 表格样式

- **表头**：深蓝底 + 白字 + 加粗（10pt）
- **数据行**：白底 + 黑字（10pt）
- **交替行**：浅灰 `#F2F2F2` 底色
- **边框**：细灰 `#DDDDDD` 线
- **对齐**：左对齐（不是居中）
- **加粗左列**（麦肯锡 KV 表格特有）：维度列加粗 + 深蓝色
- **不用**：彩行、emoji 行图标、表格内 emoj

## 5. 关键判断/重要发现（替代 callout 块）

**不用 callout 块**（粉/蓝/绿底色 + 边框）。改用：

> **判断 1.** 国外劳工保护外骨骼是最大单一细分市场，但增速并非最快赛道（医疗 45% > 工业 29.4%）。

**格式**：
- 加粗段首（"判断 1." 或 "关键判断："）—— 深蓝色
- 黑色正文（无背景色、无边框）
- 段落缩进 0.3-0.5cm

## 6. 关键数据展示

**粗体大字 + 深蓝或黑色**：

> 外骨骼总市场 2025
> **$0.56B** （18pt 深蓝粗体）
> 来源：[E10] MarketsandMarkets

**不用**：彩色 callout 块包裹关键数字。

## 7. 章节小结

每章末尾的"本节小结"段：

- 蓝色加粗段首（"本节小结"）
- 黑色正文
- 段落缩进 0.3cm
- 不加背景色、不加边框

## 8. Pitfall 清单（实战教训）

### Pitfall 1: emoj 在 macOS Word 渲染为方框 □
- **根因**：macOS Word 在中文系统下不渲染 emoj glyph
- **规避**：所有 🔴💡📌⭐⚠️🎯 等 emoj 改用文字（"关键判断："、"建议："、"小结："）
- **例外**：数据强度颜色 🔴/🟡/⚪ 仅在 Markdown 报告中允许

### Pitfall 2: 多色 callout 块显得花
- **根因**：5+ 色 callout 块（粉/蓝/绿/黄/红）破坏专业感
- **规避**：全部用"粗体段首 + 黑色正文"，视觉层级靠字号和粗细，不用色块

### Pitfall 3: 表格下方"关键洞察"段紧贴表格
- **根因**：Word 默认段落间距太小
- **规避**：表格下方若直接接段落，**必须加空行/空段**隔开；或用 paragraph_format.space_after 显式加 8-12pt

### Pitfall 4: 中文标点用半角
- **根因**：中英混排时容易用半角标点
- **规避**：中文段落用全角标点（，。；：？！），英文段落用半角

### Pitfall 5: 报告标题 vs 文件名混淆
- **报告标题**（Word 文档内）：`<主题>深度调研报告：<核心结论> (<窗口期>)` —— 决策导向
- **文件名**（文件系统）：`<主题中文>DeepResearch报告-<核心结论简写>-<窗口期>.docx` —— 机器友好
- **不要**：文件名含 v1/v2/v3（迭代在标题内重写而非文件名累积）
- **v1.3.1 强化**：详见 `references/report-title-conventions.md`——含 3 类区分（标题/文件名/Obsidian 索引）+ 实战案例 + 6 项自检

### Pitfall 6（v1.3.1 新增）：国别旗 emoj 渲染为方框
- **根因**：Pitfall 1 的特例——国别旗 emoj（🇺🇸 🇨🇳 等）也属于 emoj 字符，macOS Word 不渲染
- **规避**：用文字"美国"/"中国"或带括号形式"（美国）"/"（中国）"
- **测试案例**：AI 产品演变 .docx 第一版用了 `🇺🇸 (OpenAI) 🇨🇳 (DeepSeek)` → 检测到 16 个 emoj 字符（5 处国旗）→ 全部替换为文字"（美国）（中国）"后通过

### Pitfall 7（v1.3.1 新增）：报告有结尾页
- **根因**：习惯性加"报 告 结 束" / "本报告共 N 章 + M 附录" / "Generated by XXX" 仪式感段落
- **规避**：报告在最后一个附录（通常是"附录 D 参考来源"）自然结束即可
- **麦肯锡/BCG/贝恩咨询报告 90% 没有结尾页**——仪式感结尾页破坏专业感

## 9. Banner 图渲染（matplotlib 方案）

`scripts/word_output.py` 内部用 matplotlib 渲染深蓝科技感 banner：

- 尺寸：900x300 px（3:1 比例，适合 Word 顶部通栏）
- 背景：深蓝渐变（左 `#1F4E79` → 右 `#0E2D4A`）
- 元素：白色细网格 + 3 条数据线（白色主曲线 + 浅蓝辅线 + 金色机会线）+ 关键数据点
- 文字：左下"DEEP RESEARCH" + 主标题；右上品牌；右下关键数据（5% 渗透率 + $3.35B–$6.75B）
- 顶部金色装饰条
- 字体：PingFang SC（macOS）

**为什么用 matplotlib**：
- 跨平台、Python 原生
- 可控的配色、几何元素、文字排版
- 1350x450px PNG 嵌入 Word 不失真
- 渲染快（< 2 秒）

**不用方案**：
- WPS/Word/Pages 自动 banner（样式不可控）
- SVG（Word 兼容性差）
- CSS（无法嵌入 Word）

## 10. 完整报告结构模板

```
封面页（banner + 标题 + 元信息）
目录（中英双语）
执行摘要 / 引言
N 章正文（按场景模版）
附录 A-D（检索计划 / 证据账本 / 信源透明度 / 参考来源）
[无结尾页]
```

**不**包含：
- 仪式感"报告结束"页
- "本报告共 N 章 + M 附录"提示
- "Generated by XXX" 生成器标识（污染专业感）

## 11. 与默认 Markdown 输出的对比

| 维度 | Markdown（默认） | Word .docx（可选）|
|------|-----------------|-------------------|
| 可编辑性 | ⚠️ 排版弱 | ✅ 用户可编辑 |
| 超链接 | ✅ 可点击 | ✅ 可点击 |
| 复杂表格 | ⚠️ 需写 HTML | ✅ 底色/对齐/合并 |
| 封面 Banner | ❌ 无 | ✅ matplotlib 渲染深蓝科技感 |
| 章节封面 | ❌ 无 | ✅ 细蓝线 + 章节号 + 标题 |
| 文件大小 | 15-30KB | 60-150KB（含 banner 图）|
| 适用 | Obsidian 笔记 / 自己看 | 给领导/投资人 / 打印分享 |

## 13. 设计原则（"克制 > 丰富"）

麦肯锡年报 / 战略报告的核心视觉特征：
1. **2 色收敛**——主色深蓝 + 中性灰
2. **白底 + 蓝头条 + 灰交替**——表格干净
3. **关键判断用粗体段首**——不用 callout 块
4. **关键数字用粗体大字**——16-18pt 深蓝/黑色
5. **多留白**——段落间距 1.4-1.6
6. **去 emoj / 去 icon / 去色块**——只用文字
7. **MECE 结构**——执行摘要 → 三大议题 → 详细分析 → 启示
8. **数字突出**——粗体 + 大字 + 蓝/黑，不用红色（红色仅用于警示）

**为什么"克制" > "丰富"**：调研报告的核心是**信息密度**，不是**视觉装饰**。多色 callout 块和大图标会分散读者注意力，让读者看不清核心判断。麦肯锡/BCG/贝恩的咨询报告 90% 都是"黑白灰+1 个主色"，靠**结构和文字**而非**装饰**说服读者。

## 14. 触发方式（用户 query 示例）

```bash
"外骨骼劳工保护市场调研 --word"           # 走默认 Markdown + 额外生成 .docx
"给我一份给领导看的 Word 报告"             # 触发 Word 优先
"外骨骼调研，要 .docx 格式"                 # 触发 Word 优先
"我要打印版报告"                          # 触发 Word 优先
```

**默认行为**：如果用户没明确要 Word，走 Markdown + Obsidian 备份。Word 输出是**额外**的，不影响默认流程。

# 两院数据看板——访谈引导提纲实例（2026年6月）

## 项目背景

为两院院领导搭建管理驾驶舱和风险看板。项目组（薛海东+邹欣团队）对各中心业务理解有限，无法给出靠谱预选项，因此采用"引导思考"策略。

## 收录的消息

### 行政管理中心 · 宫茜老师

宫茜老师，您好。

打扰了。我们目前正在为两院院领导搭建一套数据看板，目的是通过数据直观呈现各领域的运行状态，帮助院领导及时发现需要关注的事项。我们团队对各部门业务了解有限，看板要呈现什么，最终需要各位院领导来定——所以想先请教您。

我们粗浅的了解是，行政管理覆盖面比较广——财务、法务、后勤、采购、人力、院办，事项似乎大致分两类：一类是预算执行、人员编制这类需要看趋势走向的；另一类是资产安全、合规这类"平时不显眼，一旦出事影响面大"的。我们对这些可能理解得不够准确，还请您指正。

如果您方便的话，想请您大致想一想以下方向——不需要详细准备，有个大致印象面谈时聊起来会更顺畅：

**关于日常需要看到的东西：**
1. 您日常最频繁过问的是哪些事项？——这些可能是看板首先需要呈现的。
2. 哪些数据您现在需要下属专门汇报才能掌握，如果看板上直接能看到会更方便？

**关于需要关注的信号：**
3. 趋势类事项里，哪些拐点您希望提前看到？比如预算执行到某个节点、人员变动的某种模式？
4. "平时不显眼但一旦出事对整个院都有影响"的领域，具体是哪些？您现在是怎么了解到这些事情的？
5. 像法务、纪检审计这类"希望永远不要启动、但一旦启动就是大事"的事项，您更希望看到什么信息——是日常的状态确认，还是只在出事时及时通知？

**关于院领导层面的关注标准：**
6. 对行政管理来说，什么标准决定一件事需要上升到院领导层面——是金额量级、事项性质，还是其他？

以上只是我们基于粗浅了解拟的大方向，您的实际判断远比我们的假设重要。近期我们再和您当面请教，感谢您的时间。

### 多部门合并示例：科学研究中心 + 战略中心 · 刘铁岩老师

刘铁岩老师，您好。

打扰了。我们目前正在为两院院领导搭建一套数据看板……您同时分管科学研究中心和战略中心，我们想就这两个方向分别请教。

**【科学研究中心】**

我们粗浅的了解是，科学研究中心的核心业务是科研项目管理和推进，项目的管理可能需要分层看待——重大项目需要逐个盯进度，一般项目需要看整体面的健康度。我们对这些可能理解得不够准确，还请您指正。

如果您方便的话，想请您大致想一想以下方向：

1. 您日常最频繁过问的是哪些项目或哪类信息？
2. "需要院领导直接盯的项目"和"看整体面就够了的项目"，您心里的边界在哪——是按经费体量、战略重要性、还是其他标准？
3. 项目运行中，什么样的信号意味着事情可能走偏——进度偏差、经费问题、人员变动，还是其他？
4. 科研成果产出方面，有没有某种模式一旦出现就意味着需要关注——比如产出节奏突然变化、某个方向长期没有进展？
5. 科研人员层面，有没有您需要关注的信号——核心人员变动、团队稳定性，还是其他？

**【战略中心】**

我们粗浅的了解是，战略中心覆盖战略研究部、公共关系部、秘书处，核心产出是院领导对外发声的内容——内参、汇报材料、演讲稿等，本质上是一个内容交付型部门。我们对这些可能理解得不够准确，还请您指正。

如果您方便的话，想请您大致想一想以下方向：

1. 您日常最频繁过问的是哪些交付物的进度？
2. 哪些在研的内参或材料，您需要随时知道进展到了哪一步？
3. 什么情况下，交付延迟或内容方向偏差会升级到需要院领导介入的程度？
4. 对外发声的内容，有没有"需要修改方向或撤回"的情形需要院领导第一时间知道？
5. 战略研究中，有没有某些议题一旦出现新动态就需要院领导关注？

以上只是我们基于粗浅了解拟的大方向，您的实际判断远比我们的假设重要。近期我们再和您当面请教，感谢您的时间。

## 设计决策记录

| 决策 | 选择 | 原因 |
|------|------|------|
| 策略 | 引导思考（非预选项） | 对各中心业务理解不足，预选项可能偏 |
| 锚点深度 | 抛出业务理解，留纠正空间 | 降低认知启动成本但不框定方向 |
| 问题数量 | 每中心6题 | 太少信息不够，太多领导不愿读完 |
| 问题分层 | 日常→信号→标准 | 由轻到重递进，降低思考启动成本 |
| 敏感词 | 按部门区分 | 不替领导定义"什么算风险" |
| 刘铁岩 | 合并为一条消息 | 避免同一人收两条重复背景的消息 |
| 邵斌 | 单独发科研 | 只分管一个中心，不需要合并 |

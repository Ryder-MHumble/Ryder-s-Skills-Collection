# B 站平台规格

## 硬约束

- **比例**: 16:9 (1920×1080) — 横屏为主
- **时长**: 1-15 分钟主流（≤ 30 分钟也接受）
- **文件大小**: ≤ 8 GB
- **格式**: MP4 (H.264/H.265)
- **帧率**: 30 fps / 60 fps
- **码率**: 推荐 8-15 Mbps（高质量内容可达 30+ Mbps）

## 内容规则

- **前 10s 决定留存**（比抖音宽松）
- **字幕**: 中文自动生成 + 创作者修正
- **字幕位置**: 画面下方 1/5 区域
- **分章节**: 强烈建议（用时间戳 + 章节标题）
- **封面**: 1080×608，自定义，标题必须清晰
- **三连引导**: 点赞/投币/收藏（在视频结尾自然引导）

## 算法偏好

1. **三连率 + 完播率 + 弹幕密度**
2. 互动率（评论）> 转发
3. 收藏率（"想再看"信号）
4. 粉丝增长贡献

## 适合的视频类型

- ✅ 深度教程
- ✅ 长讲解 / 知识分享
- ✅ 项目 demo 完整 walkthrough
- ✅ "我是怎么做 XXX 的"（过程导向）
- ✅ 系列内容

## 输出建议

- 比例锁定 16:9
- 时长 3-8 分钟最佳
- 加章节时间戳（影响推荐）
- 视频结尾 5s 内三连引导
- 封面用品牌色 + 大字标题

## 不适合

- 30s 内短内容（建议去抖音/视频号）
- 纯竖屏录制（除非用 16:9 画幅 + 主体居中）

## 与 vibe-remotion-animator 配合

B 站 16:9 横版时，建议用 1080×1920 渲染（remotion 默认）然后用 ffmpeg 转 1920×1080，或直接用 1920×1080 composition。

```bash
# ffmpeg 转换（示例）
ffmpeg -i input-1080x1920.mp4 -vf "scale=1920:1080" -c:v libx264 -crf 18 output-bilibili.mp4
```

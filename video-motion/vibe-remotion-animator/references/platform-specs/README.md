# Platform Specs — 平台规格速查

社媒视频必须在不同平台的硬约束下工作。本目录为三大目标平台（用户主战场）提供精确规格，避免做出来的视频在某个平台无法发布或体验崩坏。

## 快速对比

| 维度 | 抖音 | B 站 | 视频号 |
|---|---|---|---|
| **比例** | 9:16 (1080×1920) | 16:9 (1920×1080) | 9:16 (1080×1920) / 1:1 |
| **时长** | ≤ 3 min（最佳 15-60s） | 1-15 min 主流 | ≤ 1 min 最佳（≤ 3min） |
| **文件大小** | ≤ 128 MB | ≤ 8 GB | ≤ 256 MB |
| **格式** | mp4 (H.264) | mp4 (H.264/H.265) | mp4 (H.264) |
| **帧率** | 30 fps | 30/60 fps | 30 fps |
| **字幕** | **必加** | 中文自动 + 修正 | **必加**（静音播放场景多）|
| **首屏 hook** | 前 3s 决定生死 | 前 10s | 前 5s |
| **算法偏好** | 完播 > 互动 > 收藏 | 三连 + 完播 + 弹幕 | 社交推荐（朋友点赞）|
| **最佳场景** | 短讲解、demo 演示 | 教程、深度分享 | 简短分享、朋友圈素材 |

## 使用方式

1. **接到 brief 时**：先在 `references/scene-decomposition/routing-decision-tree.md` 里选 1-N 个目标平台
2. **在 ANALYSIS REPORT 里明确标注**：目标平台 + 比例 + 时长上限
3. **设计场景时**：按平台规格检查每个 scene 的尺寸 / 时长 / 字幕位置
4. **导出前**：跑 `scripts/check_platform_compliance.py` 验证（待 v2.1 引入）

## 跨平台策略

如果 brief 要求多平台发布：
- 优先用 9:16 竖版（覆盖抖音 + 视频号）
- B 站另出 16:9 横版
- 单视频不混比例（观众体验割裂）
- 字幕/UI 元素按各平台安全区重新排版

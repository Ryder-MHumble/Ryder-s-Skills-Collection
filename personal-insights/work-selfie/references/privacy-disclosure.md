# 隐私知情同意文案

## 完整版（首次运行展示）

```
🔮 自查知情同意

我准备从你的钉钉工作台采集以下数据，用于本次自我分析：

  📩 消息     你发送的所有消息（默认最近 30 天）
               └ 包含：文本消息、表情回应、文件消息的标题
               └ 不含：图片/视频的二进制内容

  🎙️ 听记     你发起的会议/听记列表
               └ 包含：会议标题、参会人、时长
               └ 不含：转写内容、录音

  📄 文档     你创建的钉钉文档
               └ 包含：标题、字数、最后编辑时间
               └ 不含：文档正文

  📊 表格     你创建的 AI 表格
               └ 包含：表格名、字段数、记录数
               └ 不含：记录内容

数据处理（重点）：

  ✓ 原始消息/文档只活在本次 Python 进程的内存中
  ✓ 分析完成后立即从内存释放，关闭会话即丢失
  ✓ 持久化到 skill 目录的只有"分析结果摘要"，不含任何原文
  ✓ 你的数据不会上传到任何 AI 训练集
  ✓ 野兽派×二次元名片 PNG 是临时文件，发送后立即删除

数据不会被永久保留，但你仍然可以：

  • 删除 ~/.hermes/skills/self-distill/data/last_snapshot.json
    （这会清掉历史基线，下次跑变成"全量"模式）
  • 删除 /tmp/self-distill-*.png（如果还有残留）
  • 随时撤回 dws 机器人的发消息权限

预计耗时：3-5 分钟
副作用：你的钉钉会收到 2 条消息（1 条报告 + 1 张名片）

确认开始？回复 Y 继续 / n 取消
```

## 精简版（增量运行展示）

```
🔮 增量自查

本次为增量模式：仅拉取 自 2026-05-12 以来 的新数据，与上次摘要合并。

数据范围：
  📩 消息     自上次以来新增的消息
  🎙️ 听记     自上次以来新增的会议
  📄 文档     自上次以来编辑过的文档
  📊 表格     自上次以来更新过的表格

隐私边界同上——原文不落盘，摘要写 ~/.hermes/skills/self-distill/data/last_snapshot.json

预计耗时：1-2 分钟
回复 Y 继续 / n 取消
```

## 实现要点

**1. 必须每次展示**（除非 `--skip-consent`）

**2. 用户必须明确回复 Y**（大小写不敏感）才继续
- 回复 `n` / `N` / `no` / 其他 → 立即终止，不写任何文件
- 5 分钟无回复 → 自动取消

**3. 不要"贴心"地预勾 Y**

**4. 不要"贴心"地解释为什么这些数据是安全的**——你展示边界，让用户自己判断

**5. 如果用户在跑的过程中表示担忧**：
- 立即停止后续采集
- 保留已写入的 `data/last_snapshot.json`（用户可手动删除）
- 不输出野兽派×二次元名片
- 输出一段安抚文本

## 边界场景

| 场景 | 处理 |
|------|------|
| 用户说"快点直接跑" | 不跳过，但展示精简版（2-3 句）后跑 |
| 用户说"我只查消息" | 可以，但要明示其他维度会被跳过 |
| 用户问"你会不会偷数据" | 引用完整版的"数据处理"段，**不要模糊承诺** |
| 用户回复"嗯"或"好的" | **不**视为确认，要求明确 Y |

## 法律/伦理边界

- **不分析 第三方群聊** 中「别人@你」的内容——只分析**你自己发送**的消息
- **不分析 私聊** 中「对方说」的内容——只分析**你自己发送**的消息
- 这条规则用代码强制：`collect_self.py` 过滤 `sender_user_id == self.userId` 才保留
- 如果用户主动要求"也分析别人发的"，拒绝并解释：「这超出自查的范围，且我无法保证对其他人的伦理边界」

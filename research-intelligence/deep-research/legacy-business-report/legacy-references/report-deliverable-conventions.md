# 调研报告交付规范（Ryder 偏好）

> 本文件编码 Ryder 的"调研报告交付物"硬性规范，配套 `deep-research-report` v2.0.0 § 6.1 / § 6.2 / § 7 使用。
> 任何为 Ryder 做的调研报告，都必须按此规范输出。

## 1. 标题 vs 文件名分离（强制）

**报告标题**（Word 文档内 + 文档第一行，面向读者）：
- 格式：`<主题>深度调研报告：<核心结论> (<窗口期>)`
- 例：`国内外劳工外骨骼市场深度调研报告：玩家格局、商业模式与切入机会（2026 H1）`
- 反映"决策导向"，可直接给领导/投资人看
- 副标题用括号说明时间窗或场景

**文件名**（文件系统标识，面向机器）：
- 格式：`<主题中文>-<核心结论简写>-<YYYY-MM-DD>.docx`
- 例：`国内外劳工外骨骼市场深度调研报告-玩家格局商业模式与切入机会-2026-06-10.docx`
- **不含版本号**（v1/v2/v3）—— 迭代在标题内重写而非文件名累积
- 主题词用破折号分隔核心结论简写

**反例（不要这样做）**：
- ❌ `exoskeleton-global-labor-market-2026-06-10-v3.0.md`（文件名式 + 版本号累积）
- ❌ `report-final.md` / `report-最终版.md`（无主题词）
- ❌ `国内外劳工外骨骼.md`（无核心结论简写）

**自检命令**：

```bash
ls "/Users/rydersun/Documents/Obsidian Vault/深度调研/" | grep -E 'v[0-9]|final|最终版' && echo "❌ 标题含版本号" || echo "✅ 标题规范"
```

## 2. 默认输出：单一 .docx（强制）

- **默认输出格式**：`.docx`
- 用户显式要 `.md` / `.pdf` 时例外
- **不要** 每次产出 4-5 个文件（plan/ledger/sources/verified/主报告）
- **不要** 长期累积 v1.md / v2.md / v3.md 全文

**为什么是 .docx**：
- 用户常需编辑/调整结论（vs PDF 只读）
- 可点击超链接（24 evidence_id 全部可点击跳转）
- 复杂表格样式（KV 表格 / 交替行底色 / 红色加粗突出）
- 文件小（50-100KB vs PDF 200-500KB）

**反例**：
- ❌ 主报告 .md + plan.json + ledger.md + sources.csv + verified.csv（5 个独立文件）
- ❌ 同一调研留 v1.md / v2.md / v3.md 三个版本

## 3. 工作流产物处理（强制）

`research_plan.json` / `evidence_ledger.md` / `sources.csv` / `verified_sources.csv` —— **不作为独立交付物**，**嵌入 Word 文档的附录**：

| 附录 | 来源 | 形式 |
|------|------|------|
| 附录 A | research_plan.json | KV 表格（topic / decision_question / scope / red_lines / must_cover / stop_condition 等） |
| 附录 B | evidence_ledger.md | 多列数据表（E-ID / Claim / Type / Confidence / Note） |
| 附录 C | verify_sources.py 输出 → 改写为"信源透明度报告" | 见 § 6.10 |
| 附录 D | evidence_id URL 列表 | 含可点击超链接的参考文献（python-docx `add_hyperlink`） |

**审计能力完整保留**。用户打印/分享时只打印正文 § 1-6，附录按需启用。

## 4. Obsidian 目录管理（强制）

主目录保持"主交付物 + 索引"两件：

```
深度调研/
├── <报告标题>-<核心结论简写>-2026-06-10.docx   # 主交付物（唯一）
└── <报告标题>-2026-06-10.md                     # 索引（5 句话核心判断 + 关键数据 + 链接）
```

`.md` 索引模板见 `templates/research_index.md`（标准 frontmatter + 5 条核心判断 + 关键数据 + 行动建议 + 数据边界声明）。

迭代历史移到 `.archive/` 子目录（不丢对照能力但不污染主目录）：

```
深度调研/
├── <新报告>.docx
├── <新报告>.md
└── .archive/                            # 隐藏，老版本不污染主目录
    ├── exoskeleton-v1.md
    ├── exoskeleton-v2.md
    └── exoskeleton-v3.md
```

工作文件（plan/ledger/sources/verified）放 `/tmp/<项目名>/`，**交付后清理**（保留生成脚本作为可复用模板即可）。

## 6. 视觉规范：麦肯锡商业咨询风格（强制，v5.1+）

> 这部分是 v5.1 起的硬性视觉规范。任何为 Ryder 做的 .docx 调研报告，必须符合这套视觉。

### 6.1 配色（2 色 + 中性灰，禁止彩色 callout）

| 用途 | 颜色 | 用途场景 |
|------|------|----------|
| 主色 | `#1F4E79` 深蓝 | 标题、表头、强调、关键数字 |
| 次色 | `#7F7F7F` 中性灰 | 副标题、辅助说明 |
| 浅灰 | `#F2F2F2` | 表格交替行底色 |
| 正文 | `#000000` 黑色 | 主体内容（不染色） |
| 警示 | `#C00000` 深红 | **仅**用于关键数字 + 风险预警（不滥用） |

**禁止**：
- ❌ 浅红 `#FFE6E6` / 浅蓝 `#E6F0FA` / 浅绿 `#E6F4E6` / 浅黄 `#FFF4E6` 等彩色背景块
- ❌ 多色表格行（红/绿/黄交替）
- ❌ 大段彩色文本（红色/绿色段落）

### 6.2 ❌ Emoji 禁令（macOS Word 不兼容）

**禁止任何 emoji**——macOS Word 的中文字体（SimSun）不含 emoji glyph，渲染为方框 □。

**emoji 出现的常见位置和替代方案**：

| 错误用法 | 替代方案 |
|----------|----------|
| 🔴 关键判断 | `**关键判断：**`（粗体段首） |
| 💡 提示 | `**提示：**` |
| 🎯 行动方案 | `**行动方案：**` |
| 📌 结论 | `**结论：**` |
| ⚠️ 风险 | `**注意：**` |
| 🚀 愿景 | `**愿景：**` |
| 🏢 雇主 / 👥 工会 / 🏛️ 监管 / 💼 保险 | 直接用文字（"雇主"/"工会"/"监管"/"保险"） |
| ⭐⭐⭐⭐⭐ 影响强度 | 用文字（"极强"/"强"/"中"/"弱"） |
| 🟢🟡⚪ 证据强度 | 用文字（"高"/"中"/"缺口"） |

### 6.3 ❌ 彩色 callout 块禁令

不要用以下 5 类 callout 块（v3.0 用了，被 Ryder 明确反对）：
- ❌ 粉色 callout（"关键判断"）
- ❌ 浅蓝 callout（"价值链关键洞察"）
- ❌ 浅绿 callout（"ROI 核心判断"）
- ❌ 浅黄 callout（"提示"）
- ❌ 红边框警示 callout

**替代方案 `add_insight`**：粗体段首 + 黑色正文（无背景色、无边框）：

```python
def add_insight(doc, label, text):
    """关键判断/重要发现 - 麦肯锡风格：粗体段首 + 黑色正文（无背景）"""
    p = doc.add_paragraph()
    p.paragraph_format.left_indent = Cm(0.3)
    p.paragraph_format.line_spacing = 1.5
    p.paragraph_format.space_after = Pt(6)
    run = p.add_run(f'{label} ')
    run.font.size = Pt(10.5)
    run.font.bold = True
    run.font.color.rgb = RGBColor(31, 78, 121)  # 深蓝
    run = p.add_run(text)
    run.font.size = Pt(10.5)
    run.font.color.rgb = RGBColor(0, 0, 0)      # 黑色
```

用法：`add_insight(doc, '关键判断：', '医疗是增速最快赛道，工业是最大单一细分市场...')`

### 6.4 表格规范

- 表头：深蓝 `#1F4E79` 底 + 白字 + 加粗 + 左对齐（不居中）
- 数据行：白底（奇数行）/ 浅灰 `#F2F2F2` 底（偶数行）交替
- 边框：细灰线（默认 Table Grid 样式）
- 左列（维度列）：加粗 + 深蓝字（让"维度"列视觉上更突出）
- 正文内容：黑色 10pt
- **不要**红/绿/黄行标（用文字标"高/中/低"）

### 6.5 章节封面

- 顶部 1 条细蓝色横线（6pt 粗）
- 章节号（小灰字 11pt）
- 章节标题（大蓝字 22pt，加粗）
- 副标题（中灰字 11pt，斜体）
- 底部 1 条细蓝色横线
- **不要**大色块、emoji 装饰、副标题冗长

### 6.6 关键数据展示

- 数字：粗体 16-18pt + 深蓝（不是红色，避免视觉跳跃）
- 标签：上方小灰字 9-10pt
- 来源：底部小灰字 8.5pt + 可点击超链接
- **不要**用 callout 块包裹关键数字

### 6.7 段落排版

- 行距 1.5（不是默认 1.0）
- 段落间距 4pt
- 中文用 SimSun + East Asia 字符集
- 英文/数字保持默认（避免中英文混排字距问题）
- 首行不缩进（不要用首行缩进 2 字符的传统中文格式）

### 6.8 首页 Banner 图（推荐）

用 matplotlib 渲染一张 900x300px 的深蓝科技感 banner，插入封面顶部：
- 配色：深蓝渐变 `#1F4E79` → `#0E2D4A`
- 元素：网格底纹 + 3 条数据线（白色主线 + 浅蓝辅线 + 金色机会线）+ 散点
- 文字：左下角 `DEEP RESEARCH` + 报告英文名 + 中文副标题 + 时间窗；右上角 `RYDER AI SIGNAL`；右下角关键数据（如 `5%` + `$3.35B – $6.75B`）
- 顶部装饰：1 条金色横线
- 中文字体：`PingFang SC`（matplotlib 渲染时强制指定，避免方框）

完整代码见 `templates/research_report_skeleton.py` 的 `render_banner()` 函数。

### 6.9 ❌ 结尾页禁令

**不要**在附录 D 后再加一页"报告结束 / 本报告共 N 章 / Generated by ..."——这是仪式感冗余，麦肯锡风格报告在附录 D 处自然结束。

### 6.10 附录 C 是"信源透明度报告"（v5.1+）

旧版 v3.0 把附录 C 写成"verify_sources.py 输出"（URL × HTTP × Verdict）—— 这是开发者视角的"过程态输出"，**禁止**在最终报告出现。

**v5.1+ 改写**为读者视角的"信源透明度报告"：

| 列 | 内容 | 例子 |
|----|------|------|
| 编号 | S-01 ~ S-N | `S-03` |
| 信源 | 信源名称 | `TechCrunch CES 2025` |
| 类别 | A/B/C/Social | `B（权威媒体）` |
| 用途 | 对应正文哪个 claim | `German Bionic Exia 80 磅举升（第 3 章）` |
| 状态 | 已采纳 / 备份 / 不可达 | `已采纳` |

**禁止**在附录 C 出现 "HTTP 200"、"reachable_no_keyword_hit"、"verdict: 200" 等机械预筛术语。

## 7. 5W2H 主动反问机制（强制）

**触发条件**：用户给"宽泛主题"或"几个关键词"时，**必须先反问 7 题补齐 context**，再开始检索：

```
为了把这个调研做到最贴近你真实需求，我需要先对齐几个关键点：

1. 决策目的：投资判断 / 产品立项 / 竞品对比 / 内部汇报 / 兴趣探索？
2. 受众与交付物：谁读？长度预期？输出格式（推荐 Word）？
3. 地域边界：全球 / 中国 / 美国 / 欧洲 / 特定国家？
4. 品类/场景边界：工业 To B / 消费 To C / 医疗 / 金融？包含/排除哪些场景？
5. 时间窗：最近 30 天 / 3 个月 / 12 个月 / 3-5 年趋势？
6. 重点关注维度（可多选）：市场容量 / 玩家 / 技术 / 客户 / 商业模式 / 法规 / 投融资 / 壁垒？
7. 已有材料：是否有参考资料 / 报告 / 品牌名单？
```

**反问不是形式主义**——目的是让用户参与定义问题，避免"形式完整但内容空泛"。

**何时不反问**：
- 用户已明确给出 ≥5 项 5W2H 信息
- 用户在已有报告上追问
- 用户要求"快速扫描"/"先给我看一版"（按默认假设：商业决策 / 自己+团队 / 全球 / 推断场景 / 12 个月 / Word / 麦肯锡三议题）

详细见 SKILL.md § 1。

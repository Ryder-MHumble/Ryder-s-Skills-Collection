# 场景模版：industrial-tob（工业硬件 / To B 产品调研）

> **专用于工业硬件、To B 产品、垂直行业产品的深度调研。** 与 `industry-market`（通用行业）和 `competitor`（通用对比）的关键区别：本场景强制 **scenario-fit check + employer-first 结构 + 公司格式强约束 + 雇主市场容量测算**。最典型用例：外骨骼、工业可穿戴、安全设备、企业 SaaS、医疗设备、工业机器人等。

## 适用 query 类型

- 「调研国内外 XX 工业产品/To B 产品市场」
- 「XX 行业 + XX 产品（工业级）」—— 同时含行业 + 产品
- 「帮我做 XX 产品 + 商业化路径 + 目标客户」分析
- 关键词：**工业 / 劳工 / 工地 / 物流 / 仓储 / 安全 / 企业 / To B / 工厂 / 装卸 / 搬运 / 装配线 / 工业级**（任意组合）

**触发场景的标志**（满足任意 1 条即可判定）：
- query 明确指明场景（如「劳工保护」「工厂装卸」「仓库分拣」「建筑高空」）
- query 强调工业级 / 商用 / To B（而非消费级）
- query 涉及**具体雇主画像**（如「沃尔玛」「亚马逊」「比亚迪」「某车企产线」）

## ⚠️ Step 0: Scenario-fit check（最高优先级，必做）

**目的**：在动笔前明确"什么算、什么不算"。这是 Ryder 第一次外骨骼调研明确强调的规则——"如果他不需要消费级产品，就删消费级"。

**操作**：
1. 从 query 中提取**目标场景**（如「劳工保护」「工业装卸」）
2. 列出**该场景的明确特征**（如：体力密集型 / MSD 高发 / 雇主采购 / 法规驱动）
3. **列出不属于该场景的子品类**（删除清单），例如「劳工保护」就删：
   - 消费级产品（户外徒步外骨骼、健身外骨骼）
   - 医疗康复产品（医院用步态训练外骨骼）
   - 老年助行产品（家用行走辅助）
   - 军用/特殊作业（除非 query 明确包含）
4. **生成明确的「场景边界声明」段**（写进报告开篇），例如：
   > 本报告聚焦**劳工保护 / 工业 To B 场景**的外骨骼产品。**不包含**：消费级户外外骨骼、纯医疗康复外骨骼、老年助行外骨骼、军用外骨骼。如用户后续需要其他场景，可单独调研。
5. 后续每提到一家公司，先验证其产品是否在边界内——**不在的删除**，不"留个尾巴"。

**常见错误**：
- ❌ 把"工业外骨骼"等同于"所有外骨骼"——前者是分类，后者是上位概念
- ❌ 因为一家公司有"消费级"产品就完全删除该公司——要看是否有目标场景产品，有则保留
- ❌ 调研中"顺手"加入其他场景的产品——严格遵守 query 边界

## 报告骨架（严格 3 大部分，不是 5 章）

```
# 第一部分：市场分析（雇主视角 / Market View）
## 1. 行业判断（含数据可信度颜色）
## 2. 终端雇主市场容量测算（独立板块，Walmart-style）
## 3. 法规与合规环境（仅用户指定国家/地区）

# 第二部分：国际主要企业及产品分析（产品视角 / Product View）
## 4. 重点企业深度分析（每家严格格式）

# 第三部分：趋势与不确定项
## 5. 趋势展望
## 6. 关键不确定项

# 附录
## 信源清单
```

**章节数：6 个 ##，严格 3 大部分**。**最重要规则**：第一部分全部是市场/雇主视角，第二部分全部是产品/企业视角——**不要混着写**。

## 第一部分：市场分析（雇主视角）写法

### 章节 1: 行业判断（含数据可信度颜色）

**核心要求**：
- 行业判断结论按**数据强度**标 🔴 / 🟡 / ⚪：
  - 🔴 = **多源实抓证实 + 权威机构数据**（如 MarketsandMarkets CAGR + 上市公司财报 + 多家媒体确认）
  - 🟡 = **实抓但单源 / 训练知识但多源 / 推算有据**（如只有 1 篇 TechCrunch 报道 / 训练知识截止 2025-01 但业内普遍引用）
  - ⚪ = **单源训练知识 / 反向证据不足 / 仅定性判断**（如训练知识 + 单一二手报道）
- **必须显式说明**：「以下判断的强度等级：🔴 表示...，🟡 表示...，⚪ 表示...」
- **当核心结论是 🔴 时可以"标红"声明**（如「**🔴 工业外骨骼是外骨骼最大单一细分市场，按玩家数量与应用广度判断**」）；当无法证明时诚实标 🟡 或 ⚪

**行业判断 4 要素**（每要素都给数据强度）：
1. **现状**（30 字内）
2. **核心判断**（最重要变化，30 字内）
3. **窗口期**（YYYY-MM-DD ~ YYYY-MM-DD）
4. **数据强度颜色**（🔴/🟡/⚪）

### 章节 2: 终端雇主市场容量测算（独立板块，Walmart-style）

**目的**：用**具体雇主（标杆企业）的真实数据**计算市场空间，而不是引用空洞的"市场规模 $XB"。

**方法**（详见 `references/market-sizing-methodology.md`）：
- 选 3-5 家**代表性雇主**（按用户指定地域，如美国 + 英法德）
- 对每家拆出：**总员工数 → 目标岗位数（装卸/搬运/工业劳动）→ 5% 渗透率目标用户数 → 单价 → 市场空间**
- **5% 是 illustrative**（说明性假设），不是预测——必须标「5% 渗透率下市场空间 ≈ $XM」

**Walmart 范例**（来自实战）：
```markdown
### 案例：Walmart 美国市场 [L1 实抓, 2026-06-10]

- 全球员工 2.1M，美国员工 1.6M [L1, Walmart IR, 2026-06-10]
- 美国装卸/物流岗位估算 ~240K-320K [训练知识，BLS 行业平均 15-20%]
- **5% 渗透率下** = 12K-16K 人
- 单价 $5K-$10K [训练知识，工业级均价]
- **市场空间 = $60M-$160M**（仅 Walmart 美国装卸/物流场景）
```

**分场景计算**（如果 query 涉及多场景）：
- 仓储物流：Walmart + Amazon + UPS + FedEx
- 汽车制造：通用 + 福特 + 比亚迪
- 建筑：高空作业 + 重型搬运
- 介护（边缘劳工保护）：日本/德国养老机构

### 章节 3: 法规与合规环境（**仅用户指定国家/地区**）

**核心规则**：
- **只写用户明确指定的国家/地区**——**绝不自行扩展**
- 例：用户说"美国 + 欧洲（英法德）" → 只写美/英/法/德，不"顺手"加日本/中国/韩国
- 删除任何未指定的司法管辖区（包括中国——除非用户明确要求）
- 如果用户没说 → 默认按 L1 抓取最多的国家/地区 + 显式询问用户

**每国/地区必含**：
- 关键监管机构（如美国 OSHA / 德国 DGUV / 英国 HSE）
- 核心法规（人体工程学指南、CE 认证、GDPR、PPE 法规）
- 工伤保险机制（如德国 BG 体系、Liberty Mutual）
- 重要合规标准（ANSI/NEDO、EN ISO 13849、UL 3300）

## 第二部分：国际主要企业及产品分析（产品视角）写法

### 章节 4: 重点企业深度分析

**严格顺序**（不要混进雇主/市场内容）：
- **每家公司先 2-3 句话概述**（不是公司历史，是**它在市场中的位置**）：
  > **German Bionic** - 欧洲外骨骼领导者，柏林 + 东京双总部，专注**工业 AI 外骨骼**。
- **然后按统一格式写产品/技术/价格表**：

```markdown
### 4.1 🇩🇪 German Bionic

**劳工保护产品**：

| 产品 | 材料 | 技术路线 | 助力 | 价格 | 市场份额 |
|------|------|---------|------|------|---------|
| **Exia** | 碳纤维 + 铝合金 | 有源动力 + AI 闭环 | 80 磅举升 | $5K-$10K [训练知识] | 欧洲工业外骨骼市占第一 [训练知识] |
```

**公司筛选规则**：
- 严格按 Step 0 的 scenario-fit check 执行
- 包含一家公司 = 其产品**至少 1 款**在目标场景内
- 列出 5-8 家 sweet spot（少于 5 覆盖不全；超过 10 抓取成本上升）

**表格列强制**（6 列）：
1. **产品**（名称 + 是否劳工保护场景）
2. **材料**（碳纤维/铝合金/钢/塑料/混合）
3. **技术路线**（有源/无源/AI 闭环/被动/动力外骨骼类型）
4. **关键参数**（助力/重量/续航/适用场景）
5. **价格**（区间 + 信源；用 [实抓/训练知识/缺口] 标色）
6. **市场份额**（定性 + 信源；缺数据时显式标「无公开披露」）

## 第三部分：趋势与不确定项

### 章节 5: 趋势展望

- 未来 6-18 个月 3-5 个核心趋势
- 每个趋势给：判断 + 信源 + 数据强度
- **禁止在趋势章节里夹带企业产品分析**（那是第二部分的内容）

### 章节 6: 关键不确定项

- 3-5 个目前数据不足/无法判断的点
- 单独列出，不藏在正文里
- 标明每个不确定项的"信源缺口类型"（L1 缺失 / L3 数据稀疏 / 训练知识 17 个月 gap 等）

## 信源调度（industrial-tob 专用）

### L1 必调
- 标杆雇主的**官方 IR / About / Location-Facts 页**（如 Walmart corporate.walmart.com）
- 目标公司的**官网产品页 + 定价页 + 新闻页**（如 German Bionic / SuitX / Cyberdyne）
- 监管机构**官网政策文件**（OSHA / DGUV / HSE / EU 官方）

### L2 可调（技术导向时）
- arXiv / OpenAlex 搜工业外骨骼/可穿戴相关论文
- 机构智库报告（如 MarketsandMarkets、Grand View Research）

### L3 必调
- 中文：36氪 / 晚点 / 虎嗅 / 财新
- 英文：TechCrunch / The Verge / Stratechery / Reuters / Bloomberg

### L4 可选
- KOL 行业评论（kol-interview-to-wiki 目录）
- 已监控的 Newsletter（blogwatcher 订阅）

### L5 谨慎使用
- X / Reddit / HN：**仅作为雇主/产品口碑的补充信号**，**不**作主信源
- L5 永远**不能单独证明**市场规模、收入、市占

## 强约束自检清单

- [ ] **Step 0 完成**：scenario-fit check 在开篇明示，写明"包含/不包含"
- [ ] **不混视角**：第一部分全雇主/市场视角，第二部分全产品/企业视角
- [ ] **章节数 = 6**（3 大部分下的 6 个 ##）
- [ ] **法规只写用户指定国家/地区**
- [ ] **公司筛选**：每家公司都过 Step 0 的 scenario-fit
- [ ] **公司格式**：每家先 2-3 句概述，再 6 列表格
- [ ] **价格列**：标 [实抓/训练知识/缺口] 三色
- [ ] **市场份额列**：定性 + 信源，缺数据时显式标「无公开披露」
- [ ] **市场容量测算**：至少 3 家标杆雇主，用 5% 渗透率计算
- [ ] **数据强度颜色**：🔴/🟡/⚪ 在结论处显式标注
- [ ] **至少 3 个信源层级**（L1+L3 必到）
- [ ] **L5 仅作辅助**（不能单独支撑量化判断）
- [ ] **抓取失败时显式声明**（不补训练知识伪装）
- [ ] **删除清单中的产品不出现在报告里**（如消费级、医疗级、老年助行）

## 实战教训（来自外骨骼调研首跑）

1. **scenario-fit check 是 P0**：第一次调研中"程天科技"被错分类为工业外骨骼公司（实抓后发现主营是电动轮椅）。**如果先做 scenario-fit check，会发现程天的产品线不在目标场景，删除即可**。
2. **结构混视角是最常见错误**：industry-market 模版默认 5 章都偏市场，但工业 To B 报告必须明确分"市场段"和"企业段"——否则读者读到第 4 章"监管"突然冒出公司产品，会觉得乱。
3. **法规 auto-expand 是高发错误**：用户说"美国 + 欧洲"时会"顺手"加日本/中国——必须严格按 query 范围，不"留个尾巴"。
4. **公司表格用 markdown 表格还是文字段落**？——本场景下**用 markdown 表格**（因为字段多且结构化），与 `competitor` 模版的"文字段落 Head-to-Head"不同。**选哪个**看场景：纯对比用文字，多维信息用表格。
5. **数据强度颜色不是装饰**：当用户问"是否能证明 X"时，颜色就是答案——🔴 能证明，🟡 部分能，⚪ 不能。**不要为了"显得完整"把 🟡 标成 🔴**。

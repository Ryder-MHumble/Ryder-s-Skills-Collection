# 场景模版：product-deep（单产品深度调研）

## 适用 query 类型

- 「调研一下 XX 产品」「XX 怎么样」「XX 的定价/功能/技术架构」
- 「XX 好不好用」「XX 的用户评价」「XX 的优缺点」
- 关键词：调研/产品/功能/定价/评测/体验/架构/技术细节/好不好/优缺点

## 报告骨架（严格 5 章）

```
## 1. 一句话产品画像
## 2. 核心功能与技术架构
## 3. 定价、商业模式与生态
## 4. 用户反馈与真实口碑
## 5. 优势、局限与适用场景
```

**章节数：5 个 ##，章节内不嵌套 ###**

## 章节内容指引

### 1. 一句话产品画像

- 一句话产品定位（30 字内）
- 一句话核心差异点（30 字内）
- 厂商、发布时间、当前状态（GA/Beta/Alpha）
- 格式：**定位** + **差异点** + **厂商/状态/时间**

### 2. 核心功能与技术架构

- 3-5 个核心功能（按重要性排序）
- 技术架构（如公开）：模型规模/训练数据/推理栈/部署方式
- 关键技术论文或开源仓库（如有）
- 格式示例：
  ```markdown
  **核心功能 1：Computer Use**（计算机使用） - 让 Claude 直接操作桌面 GUI 完成多步任务。技术细节：基于 Claude 3.7 多模态能力，模型输出屏幕坐标和操作指令。

  **核心功能 2：Artifacts**（工件） - 在对话中生成可交互的 HTML/SVG/React 组件。
  ```

### 3. 定价、商业模式与生态

- 定价表：免费层/Pro/Team/Enterprise 各档功能和价格
- 商业模式（订阅/按 token/开源+商业版/纯商业）
- 生态：API/SDK/插件市场/集成
- 格式：**定价档位** + **商业模式** + **生态接入**

### 4. 用户反馈与真实口碑

- 至少 3 条来自不同信源的反馈（L3/L4/L5 都可用）
- 区分「官方宣传」「专业评测」「真实用户」
- 标注反馈来源信源
- 格式：
  ```markdown
  **官方宣传视角** [L1, Anthropic 博客 2026-04-15]：Claude 3.7 在 SWE-bench 提升 30%。

  **专业评测视角** [L3, Stratechery 2026-04-22]：评测认为 Computer Use 仍存在视觉理解错误率较高问题。

  **真实用户视角** [L5, Reddit r/ClaudeAI 2026-05-01]：「用了一个月，Computer Use 帮我自动化了 80% 重复性任务，但偶尔会卡在非标准 UI 上。」

  **冲突标注**（如有）：[L1] 称完成率 95%，[L5] 用户实测 80% — 评测场景差异未明。
  ```

### 5. 优势、局限与适用场景

- 优势 3-5 条（必须有信源支撑）
- 局限 3-5 条（必须有信源支撑）
- 适用场景：明确给「适合谁」「不适合谁」
- 不确定项 1-3 条
- 格式：
  ```markdown
  **优势**：
  - 多模态能力强 [L1, L2, 2026-04-15]
  - API 稳定 [L3, 2026-04-20]

  **局限**：
  - 中文支持弱 [L5 多源, 2026-05-01]
  - 推理成本高 [L3, 2026-04-25]

  **适用场景**：开发者工具、自动化脚本、研究实验
  **不适用**：生产环境中文对话、消费级 C 端产品
  ```

## 信源调度

### L1 必调
- 厂商官网（产品页、定价页、文档、博客）
- GitHub Release（如有开源）
- 官方技术博客（如 anthropic.com/news）
- 官方 changelog/roadmap

### L2 可选
- 如产品有相关学术论文（如 Llama、Mistral），arXiv 搜
- OpenAlex 找高引产品相关工作

### L3 必调
- 中文：36氪/虎嗅 搜产品名
- 英文：TechCrunch/The Verge/The Information 搜产品名 + 时间窗口
- 评测：Stratechery、Casey Newton Platformer、Ben Thompson

### L4 可选
- kol-interview-to-wiki 搜 KOL 名字 + 产品名
- blogwatcher 已订阅的产品相关 newsletter

### L5 必调
- X 搜产品名 + 真实用户反馈
- Reddit 搜产品 subreddit
- HN 搜产品名

## 强约束自检清单

- [ ] 章节数 = 5
- [ ] 核心功能 ≥ 3 个，按重要性排序
- [ ] 定价信息有具体档位和价格（不能模糊）
- [ ] 用户反馈来自至少 3 个不同信源
- [ ] 优势/局限每条都有信源
- [ ] 适用/不适用场景明确
- [ ] 至少 3 个信源层级（L1+L3 必到）
- [ ] 真实用户反馈和官方宣传明确分开
